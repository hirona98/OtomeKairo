<!-- Block: Read Guide -->
## target と current の読み分け

- `決定済み技術選定` と `設計上の拘束` は target/current 共通の固定方針として読む
- `current 実装で接続済みの面` は、今の `browser_chat` 実装で実際に動いている範囲だけを示す
- `観測入力` と `行動出力` は、current 節に書かれた採用範囲を正本として読む

<!-- Block: Fixed Tech -->
## 決定済み技術選定

- 複数 LLM プロバイダの統一窓口は `LiteLLM` を使う
- 記憶の埋め込み検索は `sqlite-vec` を使う
- 設定インタフェースは `FastAPI` を使う
- Web サーバの実行は `Uvicorn` を使う
- `Wi-Fi の Web カメラ制御` は `ONVIF` を使う
- `音声出力用 TTS API` は `Aivis Cloud API`、`VOICEVOX Engine`、`Style-Bert-VITS2 API` を切り替えて使う

<!-- Block: Current Surface -->
## current 実装で接続済みの面

- current のブラウザ UI は `GET /` を入口に、`POST /api/chat/input`、`POST /api/chat/cancel`、`POST /api/microphone/input`、`POST /api/camera/capture`、`GET /api/chat/stream`、`GET /api/status`、`GET /api/settings/editor`、`PUT /api/settings/editor`、`GET /captures/{capture_filename}`、`GET /audio/{audio_filename}` を使う
- `POST /api/camera/observe` は API として実装済みだが、current の組み込みブラウザ UI の `Cam` ボタンはこれを呼ばず、静止画取得と次回入力への添付だけを行う
- current の観測入力は `web_input` のチャット、`source=microphone` の `microphone_message`、`camera_still_image` 添付、`self_initiated` の `camera_observation`、`browse` 後続の `network_result` に限る
- current の出力は TTS 付き `speak`、UI 向け `notice/status/message/error`、ONVIF の `look`、外部検索タスクとしての `browse` に限る
- current の `microphone` はブラウザ録音を `AmiVoice` へ同期転写し、その結果を `microphone_message` として runtime へ渡す外部接続を持つ

<!-- Block: Sensory Inputs -->
## 決定済みの観測入力

- `Wi-Fi の Web カメラ`: 視覚入力として使う。視点操作も可能にする
- `マイク入力`: 聴覚入力として使う
- `テキスト入力`: 明示的な指示や対話入力として使う

<!-- Block: Action Outputs -->
## 決定済みの行動出力

- `TTS API による音声出力`: 発話の標準出力として使い、`Aivis Cloud API`、`VOICEVOX Engine`、`Style-Bert-VITS2 API` を設定で切り替える
- `視点操作可能な Web カメラ制御`: 観測姿勢の変更として使う
- `インターネットアクセス`: 検索、取得、外部確認のために使う

<!-- Block: Control Plane -->
## 設定と操作の窓口

- 設定変更は Web サーバ経由で行う
- Web サーバは、設定変更だけでなくブラウザ経由のテキストチャット入力、カメラ静止画取得、カメラ観測起動、状態確認、SSE 配信の窓口にも使う
- ブラウザからの入力は `POST` 系 API で受け付ける
- 応答トークン、応答完了、自発メッセージ、状態通知は `SSE` でブラウザへ配信する
- Web サーバは人格コアの外側に置き、人格ループの制御面として扱う

<!-- Block: Design Constraints -->
## 設計上の拘束

- どの入力も `perception` に正規化してから人格コアへ渡す
- どの出力も `action` として明示的に計画してから実行する
- LiteLLM の利用は人格コアの外側に閉じ込め、コア本体はプロバイダ固有仕様を持たない
- sqlite-vec は記憶検索のために使い、人格の正本データは SQLite 系の保存基盤で一元管理する
- デバイス制御や API 呼び出しは `infra/` に閉じ込め、`domain/` へ漏らさない
- ブラウザ向けの継続出力は、`SSE` の読み出し専用ストリームとして扱い、ブラウザ側の受信状態を人格コアへ直接書き戻さない

<!-- Block: Immediate Build -->
## 現在実装済みの接続面

- `FastAPI + Uvicorn` による設定・チャット入力・SSE 配信サーバ
- `LiteLLM` を使った認知クライアント
- 読み取り専用の HTTP GET を使う Web 検索アダプタ
- `sqlite-vec` を使った記憶検索ストア
- `Wi-Fi Web カメラ` の ONVIF 視点操作と、ONVIF で解決した RTSP stream を `ffmpeg` で JPEG 化する静止画取得 API
- ブラウザ録音を `AmiVoice` へ送る `STT` 転写 API
- `browser_chat` 向けの設定 UI、状態表示、`SSE` ストリーム
- `Aivis Cloud API`、`VOICEVOX Engine`、`Style-Bert-VITS2 API` による発話出力
