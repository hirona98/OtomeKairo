<!-- Block: Fixed Tech -->
## 決定済み技術選定

- 複数 LLM プロバイダの統一窓口は `LiteLLM` を使う
- 記憶の埋め込み検索は `sqlite-vec` を使う
- 設定インタフェースは `FastAPI` を使う
- Web サーバの実行は `Uvicorn` を使う

<!-- Block: Sensory Inputs -->
## 決定済みの観測入力

- `Wi-Fi の Web カメラ`: 視覚入力として使う。視点操作も可能にする
- `マイク入力`: 聴覚入力として使う
- `テキスト入力`: 明示的な指示や対話入力として使う

<!-- Block: Action Outputs -->
## 決定済みの行動出力

- `TTS による音声出力`: 発話の標準出力として使う
- `視点操作可能な Web カメラ制御`: 観測姿勢の変更として使う
- `インターネットアクセス`: 検索、取得、外部確認のために使う
- `API を使用した SNS アクセス`: 外部サービス上での観測と行動に使う
- `API を使用した LINE 通知`: 明示的な通知行動として使う

<!-- Block: Control Plane -->
## 設定と操作の窓口

- 設定変更は Web サーバ経由で行う
- Web サーバは、設定変更だけでなくテキスト入力や状態確認の窓口にも使う
- Web サーバは人格コアの外側に置き、人格ループの制御面として扱う

<!-- Block: Design Constraints -->
## 設計上の拘束

- どの入力も `perception` に正規化してから人格コアへ渡す
- どの出力も `action` として明示的に計画してから実行する
- LiteLLM の利用は人格コアの外側に閉じ込め、コア本体はプロバイダ固有仕様を持たない
- sqlite-vec は記憶検索のために使い、人格の正本データは SQLite 系の保存基盤で一元管理する
- デバイス制御や API 呼び出しは `infra/` に閉じ込め、`domain/` へ漏らさない

<!-- Block: Immediate Build -->
## 先に実装する接続面

- `FastAPI + Uvicorn` による設定・テキスト入力サーバ
- `LiteLLM` を使った認知クライアント
- `sqlite-vec` を使った記憶検索ストア
- `Wi-Fi Web カメラ` と `マイク` の観測入力
- `TTS` による発話出力
