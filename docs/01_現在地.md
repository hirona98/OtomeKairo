# 現在地

## この文書の役割

この文書は、OtomeKairo の現行実装がどこまで到達しているか、何をまだ持っていないか、どの制約を抱えているかを管理する。

- 長く残る意味境界は `docs/design/` を正とする
- 外向き API wire 契約は `docs/design/api/` を正とする
- 記憶 subsystem の詳細契約は `docs/design/memory/` を正とする
- 関数分割や処理順の最終正本は `src/` のコードとする
- 一時的な実装計画や次の作業列挙は、この文書の責務に含めない

## 実装済み範囲

現行コードで到達している主要範囲は次である。

- HTTPS API サーバとして bootstrap、会話入力、wake、`status`、`config` 系の選択と CRUD、catalog、inspection、`events/stream`、`logs/stream`、`PATCH /api/capabilities/{capability_id}/state` を持つ
- route surface は `docs/design/api/` の現行ファミリーへ対応しており、差分の焦点は route 欠落ではなく wire 詳細と追加検証にある
- 通常会話、`wake`、capability result follow-up が共有判断パイプラインへ合流する
- 共有判断パイプラインでは、必要時の画像観測要約、`RecallHint`、`RecallPack`、answer contract / evidence resolution、`world_state` 更新、initiative 文脈、decision / reply / capability request を扱う
- `wake_policy.observations` を持ち、interval wake では enabled observation を取得して `world_state` へ反映してから 1 回だけ wake 判断を行う
- `turn consolidation`、background memory postprocess worker、`vector_index_sync`、`reflective consolidation` を持つ
- capability request dispatch と result follow-up を持ち、`vision.capture`、`external.status`、`schedule.status`、`device.status`、`body.status`、`environment.status`、`location.status`、`social.status` の concrete result endpoint が開いている
- process startup で orphaned `waiting_result` の `ongoing_action` を掃除し、新しい capability request を塞がない
- 回帰確認の入口として `smoke` と `real-llm-smoke` の既存手順を持つ

## 未実装範囲と既知の制約

現時点では次を前提に扱う。

- `docs/design/api/` には完成形の wire 契約を含むため、設計済みだがコードへ未反映の項目がありうる。厳密な実装範囲はこの文書と `src/` を併せて判断する
- 現行フロー専用の補助文書は置かず、内部フローの把握はコードを正とする
- `/api/events/stream` は client -> server message として `hello` だけを受け付ける。`vision.capture` が accepted された client は、`hello.vision_sources` に少なくとも 1 件の source を載せ、各 `vision_source_id` は `vision_source:` で始める必要がある
- `/api/conversation` の `images` は image Data URI だけを受け付け、最大 1 件に制限する
- `/api/capability/result` は capability family ごとに `result` の必須 field を追加検証する
- `vision.capture` は raw image payload、OCR 全文、UI 構造化を保存しない
- capability dispatcher は、配送先 binding が一意に定まる capability だけを実行対象にする
- `pending_intent_candidates` は runtime-only で保持し、永続化しない
