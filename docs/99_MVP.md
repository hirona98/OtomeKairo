# MVP

## この文書の位置づけ

この文書は、OtomeKairo の現在の MVP を定義する。
長く残る設計判断は `docs/design/` 配下を正とし、この文書では「今どこまでを作るか」「何を成功とみなすか」を定める。

## MVP の範囲

最初の MVP は、会話を主接点として、記憶と現在の個を成立させる核を作ることに集中する。
ただし、内部設計を会話専用ボットへ寄せることはしない。

## MVP で含めるもの

- `CocoroConsole` を唯一のクライアントとして接続する
- OtomeKairo を常駐サーバとして稼働させる
- ユーザー文字入力と軽量なクライアント文脈を観測として受け取る
- 会話を継続し、日をまたいだ記憶を保持する
- 自発的に会話提案できる
- ユーザーの嗜好を学習できる
- 同一人格設定のまま複数の記憶を切り替えて検証できるように、人格設定と記憶を独立に切り替えられる

## MVP で主対象にしないもの

- カメラ、移動手段、Web アクセスなどの外界行動そのもの
- 人間が使うあらゆる接点への展開
- 細かな安全制御や権限制御

ただし、将来の拡張を見据えて、会話以外の行動も収容できる設計方針はこの段階で定める。

## 成功条件

MVP の成功条件は次の 3 点とする。

- 数日またいでも、別セッションで過去を踏まえた会話ができる
- 自分から次の行動や会話を提案できる
- ユーザーの嗜好を学習し、それが会話に反映される

## 非目標

現段階では、次を目標にしない。

- すべての行動能力を一度に実装すること
- 人間と同等の身体性を先に作ること
- 実装都合で仕様を先に決めること
