# 開発者設定仕様

<!-- Block: Purpose -->
## このドキュメントの役割

- このドキュメントは、`config/developer.toml` の固定仕様を定義する正本である
- 対象は開発者だけが手で編集する起動時設定であり、Web UI や SQLite の `runtime_settings` には載せない
- 目的は、`logger` の level や `LiteLLM` の詳細ログのような開発用観測設定を、コード編集なしで切り替えられるようにすることにある
- `log/otomekairo.log` のローテーション条件は、このファイルではなく実装側の固定仕様として扱う

<!-- Block: Scope -->
## このドキュメントで固定する範囲

- 固定するのは、ファイルの保存場所、読み込みタイミング、必須キー、型、検証規則、適用先である
- 固定しないのは、人格設定、モデル設定、API キー、UI から編集される設定である

<!-- Block: File Placement -->
## ファイル配置

- パスは `config/developer.toml` に固定する
- 文字コードは `UTF-8` に固定する
- 改行は `LF` に固定する
- このファイルはリポジトリに含め、起動時に必ず存在していなければならない

<!-- Block: Lifecycle -->
## 読み込みと適用のタイミング

- `run_all.py`、`run_web.py`、`run_runtime.py` は、それぞれ `configure_process_logging(...)` を呼ぶ前に `config/developer.toml` を読み込む
- 適用タイミングは起動時だけに固定する
- 起動中の自動再読込は行わない
- 読み込んだ値は SQLite へ保存しない
- Web API から参照・更新しない
- `log/otomekairo.log` は起動後の全 process で共有し、約 1MiB ごとに 5 世代ローテーションする

<!-- Block: Validation -->
## 検証規則

- `schema_version` は必須で、現在は `1` に固定する
- 未知の top-level key、未知の section、未知の process 名は起動失敗にする
- 型不一致、列挙外の log level、TOML 構文エラーは起動失敗にする
- `process.<name>.logging.loggers` には任意の Python logger 名を書いてよい
- `process.<name>.logging.loggers` に `LiteLLM` と `litellm` は書いてはならない
- `LiteLLM` の level は `integrations.litellm.log_level` だけで決める

<!-- Block: Log Levels -->
## 許可する log level

- `CRITICAL`
- `ERROR`
- `WARNING`
- `INFO`
- `DEBUG`
- `NOTSET`

<!-- Block: Schema -->
## ファイルスキーマ

- top-level key は `schema_version`、`process`、`integrations` の 3 つに固定する
- `process` 配下には `launcher`、`web`、`runtime` の 3 つを必須で持つ
- 各 process は `logging` table だけを持つ
- 各 `logging` table は `root_level`、`console_level`、`file_level`、`loggers` の 4 つを必須で持つ
- `integrations` 配下には `litellm` を必須で持つ
- `integrations.litellm` は `log_level` だけを持つ

<!-- Block: Semantics -->
## 各キーの意味

- `process.<name>.logging.root_level`
  - root logger 全体の level を決める
- `process.<name>.logging.console_level`
  - 端末へ出す handler の level を決める
- `process.<name>.logging.file_level`
  - `log/otomekairo.log` とそのローテーション世代へ出す handler の level を決める
- `process.<name>.logging.loggers`
  - `uvicorn` や `httpx` など個別 logger の level を process ごとに固定する
- `integrations.litellm.log_level`
  - `LITELLM_LOG` 環境変数と `LiteLLM` / `litellm` logger level の両方へ反映する

<!-- Block: Security -->
## セキュリティ上の固定ルール

- 秘密情報を平文でログへ出してはならない
- `api_key`、`Authorization: Bearer ...`、`token`、`password`、`bot_token` に該当する値は、formatter 側で常に mask する
- `data:*;base64,...` を含む BASE64 本文と、長い BASE64 文字列は formatter 側で常に省略する
- 秘密情報の mask は `config/developer.toml` から無効化できない

<!-- Block: Example -->
## 既定ファイル

```toml
schema_version = 1

[process.launcher.logging]
root_level = "DEBUG"
console_level = "INFO"
file_level = "DEBUG"

[process.launcher.logging.loggers]
uvicorn = "INFO"
"uvicorn.error" = "INFO"
"uvicorn.access" = "INFO"
"py.warnings" = "WARNING"
asyncio = "INFO"
httpcore = "WARNING"
httpx = "WARNING"
openai = "INFO"
zeep = "WARNING"

[process.web.logging]
root_level = "DEBUG"
console_level = "INFO"
file_level = "DEBUG"

[process.web.logging.loggers]
uvicorn = "INFO"
"uvicorn.error" = "INFO"
"uvicorn.access" = "INFO"
"py.warnings" = "WARNING"
asyncio = "INFO"
httpcore = "WARNING"
httpx = "WARNING"
openai = "INFO"
zeep = "WARNING"

[process.runtime.logging]
root_level = "DEBUG"
console_level = "INFO"
file_level = "DEBUG"

[process.runtime.logging.loggers]
uvicorn = "INFO"
"uvicorn.error" = "INFO"
"uvicorn.access" = "INFO"
"py.warnings" = "WARNING"
asyncio = "INFO"
httpcore = "WARNING"
httpx = "WARNING"
openai = "INFO"
zeep = "WARNING"

[integrations.litellm]
log_level = "DEBUG"
```

<!-- Block: Current Decision -->
## このドキュメントで確定したこと

- 開発者専用の起動時設定は `config/developer.toml` を正本にする
- `runtime_settings` や設定 UI には混ぜない
- `LiteLLM` のログ level は `integrations.litellm.log_level` で制御する
- `zeep` の内部 WSDL / XSD debug は既定で `WARNING` に固定して、通常運用ログへ出さない
- 構文エラーや未知キーは silent に無視せず、必ず起動失敗にする
