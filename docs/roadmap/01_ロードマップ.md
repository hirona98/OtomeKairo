## 進め方の前提

ここでは、すでに `design/06_設定と状態.md` から `design/13_自発判断と自発行動.md` までで固めた基礎設計の上に残る未解決論点だけを書く。

また、次を前提に進める。

- `CocoroConsole` は設定の正本ではなく、編集 UI として振る舞う
- 記憶設計の主線は `docs/design/memory/` にあり、ここには未解決の次段だけを書く
- 通常 API の 4 面と接続 bootstrap の境界はすでに正本 docs へ移してある
- MVP の詳細設計を止めている論点から先に閉じる

<!-- Block: Step1 -->
## 1. 記憶設計の次段

記憶設計の主線は `docs/design/memory/` を正としつつ、次段で検討する項目だけをここに残す。

ここで詰めること:

- 複合意図の同時最適化
- 直近窓の外にある曖昧参照
- 長期変化の分析をどの層で扱うか
- 正確引用や詳細時系列を責務に含めるか
- 複雑な外界行動の実行設計をどこから別責務に切り出すか

最後の項目は、`design/13_自発判断と自発行動.md` と連動して詰める。

<!-- Block: Step2 -->
## 2. 将来の身体性と外界接続

会話以外の行動能力をどう設計に収めるかは、中長期の拡張性に関わる。

ここで詰めること:

- 観測能力と作用能力をどう分けるか
- 能力追加時に人格や記憶とどう接続するか
- 人間互換に近づけるための拡張順序

<!-- Block: Gate -->
## 詳細設計へ進んでよい条件

少なくとも次が揃った時点で、詳細設計へ進む。

- 応答や自発判断の根拠をどの粒度で追うか説明できる
- `reply / noop / future_act` の判断境界と `future_act` 候補キューの扱いを説明できる
- 記憶設計の次段で何を MVP に含め、何を後段へ送るか説明できる
- 段階トレースで何を保存し、何を保存しないか説明できる
- 接続 bootstrap と通常 API の境界が docs 全体で一致している
- `CocoroConsole` は編集 UI であり、設定の正本ではないと docs 全体で一致している
