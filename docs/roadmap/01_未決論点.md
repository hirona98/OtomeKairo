## 論点 1: OtomeKairo 専用 API の概念整理

`CocoroGhost` 互換ではなく、OtomeKairo に自然な API をどう切るかは今後の重要論点である。

ここで詰めるべきこと:

- サーバが返す応答と状態の種類
- 人格切り替えや記憶切り替えの操作概念

<!-- Block: Persona -->
## 論点 3: 人格設定の粒度

人格設定ファイルに何を含めるべきかは、会話品質と保守性の両方に影響する。

ここで詰めるべきこと:

- 性格と口調の関係
- 行動判断傾向の持たせ方
- 人格更新の扱い

<!-- Block: Models -->
## 論点 4: モデル役割の具体的な分け方

LiteLLM を前提に、役割ごとにモデルを分けられるようにするが、その境界は今後詰める。

ここで詰めるべきこと:

- 会話と判断を同じモデルにするか
- 想起入口と行動判断を分けるか
- コストと品質の折り合い

<!-- Block: Debug -->
## 論点 5: デバッグ可能性の設計

OtomeKairo は長期伴走を目指すため、後から挙動を検証できることが重要である。

ここで詰めるべきこと:

- 応答や行動の根拠追跡の粒度
- 保存量と見通しやすさの両立
- 観測ログと判断ログの分け方

<!-- Block: Initiative -->
## 論点 6: 自発行動の扱い

自発行動は本プロジェクトの特徴のひとつである一方、設計上の難所でもある。

ここで詰めるべきこと:

- `reply / act / noop` の閾値
- 文脈と頻度の関係
- 自発会話から外界行動へ広げる順序
- 複雑な外界行動の実行設計をどこから別責務に切り出すか

<!-- Block: Embodiment -->
## 論点 7: 将来の身体性と外界接続

会話以外の行動能力をどう設計に収めるかは、中長期の拡張性に関わる。

ここで詰めるべきこと:

- 観測能力と作用能力をどう分けるか
- 能力追加時に人格や記憶とどう接続するか
- 人間互換に近づけるための拡張順序
