# 目的とMVP

## 目的

OtomeKairo の目的は、単なる対話ボットではなく、長期にわたって人間と伴走できる自律的な存在を作ることである。

ここでいう「長期伴走」とは、少なくとも次を含む。

- 日をまたいでも会話と関係性が継続する
- 過去のやりとりや嗜好を踏まえて応答できる
- 必要に応じて、自発的に次の行動や会話を提案する
- ユーザー以外の人や出来事も認識し、世界を自分なりに理解していく

## 目指す存在像

OtomeKairo は、ユーザー専用の道具ではなく、個として存在する AI を目指す。
ただし、最初の対象ユーザーは開発者本人であり、そこから設計を育てる。

この存在像には次の性質を含む。

- 自分と他者を区別できる
- 継続する人格を持つ
- 記憶によって考え方や反応が変わる
- 会話だけでなく、将来的には外界への行動判断も扱う

## MVP の範囲

最初の MVP は、会話を中心とした長期伴走の核を成立させることに集中する。

MVP で含めるもの:

- `CocoroConsole` を唯一のクライアントとして接続する
- OtomeKairo を常駐サーバとして稼働させる
- ユーザー文字入力と軽量なクライアント文脈を観測として受け取る
- 会話を継続し、日をまたいだ記憶を保持する
- 自発的に会話提案できる
- ユーザーの嗜好を学習できる
- 同一人格のまま複数の記憶を切り替えて検証できるように、人格と記憶を独立に切り替えられる

MVP でまだ主対象にしないもの:

- カメラ、移動手段、Web アクセスなどの外界行動そのもの
- 人間が使うあらゆる接点への展開
- 細かな安全制御や権限制御

ただし、将来の拡張を見据えて、会話以外の行動も収容できる設計方針はこの段階で定める。

## 成功条件

MVP の成功条件は次の 3 点とする。

- 数日またいでも、別セッションで過去を踏まえた会話ができる
- 自分から次の行動や会話を提案できる
- ユーザーの嗜好を学習し、それが会話に反映される

## 非目標

現段階では、次を目標にしない。

- すべての行動能力を一度に実装すること
- 人間と同等の身体性を先に作ること
- 実装都合で仕様を先に決めること
