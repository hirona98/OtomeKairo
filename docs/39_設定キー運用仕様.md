# 設定キー運用仕様

<!-- Block: Purpose -->
## このドキュメントの役割

- このドキュメントは、`POST /api/settings/overrides` で受け付ける設定キー、型、適用範囲、基本制約を固定する正本である
- 設定UIの全体保存モデルは `docs/42_設定UI仕様.md` を正本とし、このドキュメントは scalar な設定キー更新 API だけを扱う
- `retrieval_profile` や `motion.animations` のような構造化値は、この API では受け付けない
- 実際の JSON 形は `docs/36_JSONデータ仕様.md`、Web API の振る舞いは `docs/35_WebAPI仕様.md` を見る

<!-- Block: Core Rules -->
## 共通原則

- `settings_overrides.key` は、このドキュメントに列挙したキーだけを受け付ける
- 未登録キー、型不一致、範囲違反、列挙違反、`apply_scope` 不一致は Web サーバが `400 Bad Request` で拒否する
- `requested_value` は `string`、`integer`、`number`、`boolean` だけを受け付ける
- `object` と `array` は受け付けない
- 秘密情報も登録キーであれば通常の設定値として受け付ける
- `apply_scope="runtime"` は同じ短周期で `runtime_settings` に反映する
- `apply_scope="next_boot"` は `settings_overrides` に保存して、次回起動時に materialize する

<!-- Block: Error Codes -->
## エラーコード

- 未登録キー: `unknown_settings_key`
- `apply_scope` 不一致: `invalid_settings_scope`
- 型違反、範囲違反、列挙違反: `invalid_settings_value`

<!-- Block: Enumerations -->
## 固定列挙

- `llm.reasoning_effort`: `""`、`low`、`medium`、`high`
- `behavior.response_pace`: `careful`、`balanced`、`quick`
- `behavior.proactivity_level`: `low`、`medium`、`high`
- `behavior.browse_preference`: `avoid`、`balanced`、`prefer`
- `behavior.notify_preference`: `quiet`、`balanced`、`proactive`
- `behavior.speech_style`: `gentle`、`neutral`、`firm`

- `speech.tts.provider`: `aivis-cloud`、`voicevox`、`style-bert-vits2`
- `speech.tts.aivis_cloud.output_format`: `wav`、`mp3`、`ogg`、`aac`、`flac`
- `speech.stt.provider`: `amivoice`
- `integrations.notify_route`: `ui_only`、`discord`

<!-- Block: Runtime Rejection -->
## ランタイムで reject してよい条件

- 設定キーとしては正しいが、現在の実行環境では適用不能
- Discord 通知を選んだが、必要な認証情報が実行段で不正
- TTS / STT プロバイダ設定が実環境の接続先と矛盾する
- `next_boot` 前提の設定を現在サイクルへ即時反映しようとしている

<!-- Block: Fixed Decisions -->
## このドキュメントで確定したこと

- scalar な設定キー更新 API で扱うキー集合は `src/otomekairo/schema/settings.py` の登録値に一致する
- 会話、記憶、キャラクター、TTS、STT、モーション、通知のキーは OtomeKairo 側へ統合して扱う
- 構造化値は設定UIの全体保存で扱い、`POST /api/settings/overrides` では扱わない
