# 人格と記憶

## 基本原則

OtomeKairo では、人格設定と記憶を別の層として扱う。

- 人格設定は、どう考え、どう振る舞い、何を重視するかの基底である
- 記憶は、何を経験し、何を理解し、誰や世界とどう関わってきたかの蓄積である
- 現在の個は、人格設定と記憶からその時点で成立する判断主体である

実際の挙動は、人格設定と記憶から組み上がる現在の個の文脈で決まる。
人格設定だけでも、記憶だけでも、OtomeKairo の現在の個は定まらない。

## この文書と `memory/` 配下の関係

この文書は、人格設定、記憶、現在の個の意味境界を定める上位文書である。
`memory/` 配下は、この前提を変えずに記憶 subsystem の詳細を定める。

- `04_人格と記憶.md`
  - 上位概念と責務境界を定める
- `memory/00_記憶設計方針.md`
  - 記憶 docs 全体の入口と読み方を示す
- `memory/01_記憶システム概要.md` から `memory/04_記憶更新と再整理.md`
  - 記憶の正本契約を定める

記憶の内部構造、想起、更新、再整理の詳細は `memory/` 配下を正とする。

## 人格設定

人格設定は、持続的な行動傾向を与える設定である。
人格設定の内部粒度は `09_人格設定詳細.md` を正とする。

人格設定には少なくとも次の要素が含まれる。

- 基本姿勢
- 表現傾向
- 価値観
- 判断傾向
- 他者との関わり方
- 行動時の姿勢

人格設定は、経験のたびに全面的に書き換えるものではない。
経験から育つ具体的な自己理解や関係理解は、主に記憶側で扱う。

## 記憶

記憶は、経験の蓄積とそこから育った理解を保持する層である。

記憶には少なくとも次のものを含む。

- 生の出来事
- 経験のまとまり
- 人物、対象、場所、出来事の継続理解
- 関係
- 未完了や継続中の事柄
- 感情モデル
- 世界理解

記憶の正本領域は `events / episodes / memory_units` と感情モデルで構成する。
感情モデルの詳細は [16_感情モデル.md](./16_感情モデル.md) を正とする。

記憶は単なる履歴ではない。
現在の個の文脈を変化させ、次の判断や振る舞いに効く材料として扱う。

## 派生する志向状態（`drive_state`）

`drive_state` は、人格設定と記憶から派生する中期内部状態である。
`drive_state` は、次の数回から数日の判断で、何を気にかけ、どちらへ向きやすいかを持つ。

これは人格設定の代替でも、記憶の代替でもない。

- 人格設定
  - どう考え、何を重視するかの基底を持つ
- 記憶
  - 経験と理解の蓄積を持つ
- `drive_state`
  - その時点で前に出やすい中期の向きを持つ

`drive_state` の意味境界は `05_判断と行動.md` と `06_状態モデル.md` を正とする。

## 人格設定、記憶、現在の個の関係

人格設定、記憶、現在の個は、同じものではない。

- 人格設定
  - 持続的な基底定義である
- 記憶
  - 経験の蓄積と、そこから育った具体的な理解である
- `drive_state`
  - 人格設定と記憶から派生する中期の志向状態である
- 現在の個
  - その時点の人格設定と記憶から成立する判断主体である

ここで重要なのは、`現在の個` を独立した保存物として扱わないことである。
長く残る基底の正本は人格設定と記憶であり、`drive_state` や保留意図のような派生状態は別の状態層として持つ。
現在の個は判断時に都度立ち上がる。

判断時の関係は次のように整理する。

- 人格設定は、どう考え、何を重視し、どう振る舞うかの基底を与える
- 記憶は、誰をどう知っているか、何が継続中か、過去に何があったかという具体性を与える
- `drive_state` は、人格設定と記憶から派生し、今しばらく何へ向かいやすいかを与える
- 現在の個は、その回の観測、必要な状態、人格設定、記憶をまとめて成立する
- 現在の判断は、その回に成立した現在の個から生まれる
- 判断後に保存するのは、出来事、記憶更新、中期内部状態、短期内部状態であり、現在の個そのものではない

## ライフサイクル

OtomeKairo の現在の個の連続性は、人格設定と記憶の双方で支えられる。

- 人格設定の変更は、明示的に行われる
- 記憶の更新は、経験に応じて継続的に起こる
- `drive_state` は、人格設定と記憶の変化に応じて再構築される
- 人格設定の変更は記憶を自動で書き換えない
- 記憶セットの切り替えや初期化は人格設定を自動で書き換えない

## 切り替えと更新の境界

人格設定変更と記憶変更は、別の意味操作として扱う。

- 一方の変更が他方を暗黙に変えない
- 一時的な作業文脈や `drive_state`、保留状態は、新しい組み合わせから再構築する
- 人格設定の削除や更新は、人格設定そのものを対象にする
- 記憶セットの削除や初期化は、その記憶セット全体を対象にする

個別の `event`、`episode`、`memory_unit` を標準操作として削除しない。

選択値の保持や設定変更の詳細は `07_設定モデル.md` と `08_設定変更.md` を正とする。

## 世界認識の前提

OtomeKairo は、ユーザー本人だけを相手にする存在ではない。
世界を自分なりに理解するため、少なくとも次を扱える必要がある。

- 自分
- 個々の人物
- 物、場所、出来事
- それらの関係

## デバッグと観測

人格設定、記憶、現在の個の変化は、後から追跡できる必要がある。

- 生の出来事を辿れること
- どの判断と更新が起きたかを辿れること
- 何を根拠に理解や関係が変わったかを見返せること

デバッグ記録の保証内容は `13_デバッグ可能性.md` を正とする。
