# 対話画像入力

## 目的

通常会話 API が、`text` と `client_context` だけでなく会話添付 image payload も受け取る。

この段階では raw image payload をそのまま判断や永続化へ流さない。
`input_interpretation` により詳細な視覚説明へ変換し、共有判断パイプライン、想起、記憶整理へ渡す。
会話添付画像は `vision.capture` の非同期 capability result ではない。
desktop client、capability request、ongoing_action、capture timeout とは結び付けない。
LLM prompt でのユーザー発話と内部視覚文脈の分離方針は `27_プロンプト文脈分離方針.md` を正とする。

## 境界

この機能で行うこと:

- `POST /api/conversation` が任意の `images` を受け取る
- 画像を詳細な視覚説明へ変換する
- 観測要約を `recall_hint / RecallPack / decision / speech` の入力へ反映する
- raw image payload を保存せず、詳細な視覚説明を `visual_observation_record` として残す

禁止すること:

- `text` を任意化する
- OCR 全文抽出
- UI 要素や座標の構造化
- raw 画像の長期保存
- 音声や動画入力
- 添付画像だけから `world_state.visual_context` を更新すること

## API 形状

`POST /api/conversation` の request body は次を受ける。

```json
{
  "text": "この画面どう見える？",
  "images": ["data:image/png;base64,..."],
  "client_context": {
    "source": "CocoroConsole",
    "client_id": "console-...",
    "active_app": "Browser",
    "window_title": "docs | Browser",
    "locale": "ja-JP"
  }
}
```

- `text` は従来どおり必須の string
- `images` は任意の Data URI 配列
- 1 回の通常会話で扱う画像は 1 件に固定する
- `client_context` は任意の object

## パイプライン

通常会話で `images` がある場合は、共有判断パイプラインへ入る前に画像観測要約を行う。

1. request body を検証する
2. image payload を詳細な `summary_text` へ変換する
3. `observation_summary.source=conversation_attachment` と `image_interpreted / visual_summary_text / visual_confidence_hint` を入れる
4. `visual_summary_text` を `visual_observation_record` として保存する
5. 検索・想起用の `augmented_query_text` に観測要約を足す
6. 永続化する user input text は元の `text` を保つ

## 保存境界

- raw image payload は保存しない
- user event の `text` は元の `text` を保存する
- `visual_observation_record` には詳細な視覚説明を残す
- `cycle_trace.input_trace.observation_summary` には視覚説明を残す
- `cycle_trace.input_trace.input_context_addition_summary` と `decision_trace.input_context_addition_summary` には、添付画像要約として内部補助文脈へ追加した差分だけを残す
- `cycle_trace.input_trace.augmented_query_summary` と `decision_trace.augmented_query_summary` には、検索・想起用に拡張した文字列の短縮表示を残す
- `input_interpretation`、`decision_generation`、`expression_generation` の `visual_observation_context` には、解釈済み会話添付画像の要約を渡す
- `world_state` 更新 source pack には会話添付画像の `visual_summary_text` を渡さない

`internal_context.visual_observation_context.source=conversation_attachment` かつ `image_interpreted=true` の場合、会話添付画像は `visual_summary_text` として解釈済みである。
`decision_generation` は raw image が prompt に無いことを理由に画像欠落扱いしない。
添付画像要約だけで断定できない場合は、`noop` ではなく短い確認を含む `speech` を選ぶ。

## failure の扱い

- `images` が配列でない、空文字要素を含む、Data URI でない、または 2 件以上の場合は `400 invalid_images`
- 画像観測要約が失敗した場合は cycle を `internal_failure` として残す
- silent fallback で画像なし会話へ丸めない
