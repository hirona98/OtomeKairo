# 対話画像入力

## 目的

通常会話 API が、`text` と `client_context` だけでなく image payload も受け取れるようにする。

この段階では raw image payload をそのまま判断や永続化へ流さない。
`input_interpretation` により短い観測要約へ変換し、共有判断パイプラインへ渡す。

## 境界

この段階で行うこと:

- `POST /api/conversation` が任意の `images` を受け取る
- 画像を短い観測要約へ変換する
- 観測要約を `recall_hint / RecallPack / decision / reply / world_state` の入力へ反映する
- raw image payload を保存せず、inspection には短い要約だけを残す

この段階で行わないこと:

- `text` を任意化する
- OCR 全文抽出
- UI 要素や座標の構造化
- 画像の長期保存
- 音声や動画入力

## API 形状

`POST /api/conversation` の request body は次を受ける。

```json
{
  "text": "この画面どう見える？",
  "images": ["data:image/png;base64,..."],
  "client_context": {
    "source": "CocoroConsole",
    "client_id": "console-...",
    "active_app": "Browser",
    "window_title": "docs | Browser",
    "locale": "ja-JP"
  }
}
```

- `text` は従来どおり必須の string
- `images` は任意の Data URI 配列
- `client_context` は任意の object

## パイプライン

通常会話で `images` がある場合は、共有判断パイプラインへ入る前に画像観測要約を行う。

1. request body を検証する
2. image payload を短い `summary_text` へ変換する
3. `observation_summary` に `image_interpreted / visual_summary_text / visual_confidence_hint` を入れる
4. LLM に渡す有効入力へだけ観測要約を足す
5. 永続化する user input text は元の `text` を保つ

## 保存境界

- raw image payload は保存しない
- user event の `text` は元の `text` を保存する
- `cycle_trace.input_trace.observation_summary` には短い観測要約を残す
- `world_state` 更新には `visual_summary_text` を補助根拠として渡してよい

## failure の扱い

- `images` が配列でない、または空文字要素を含む場合は `400 invalid_images`
- 画像観測要約が失敗した場合は cycle を `internal_failure` として残す
- silent fallback で画像なし会話へ丸めない
