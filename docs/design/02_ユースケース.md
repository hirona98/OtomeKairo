# ユースケース

## この文書の位置づけ

OtomeKairo が何をすべきかを、抽象概念ではなく具体的な利用場面で確認するための文書である。
ここでは動作の細部ではなく、体験として何が起きるべきかを定義する。

## ユースケース 1: 日をまたいだ会話継続

ユーザーが前日に話した内容を、翌日 OtomeKairo が自然に踏まえて会話する。

期待する状態:

- 過去の会話が単なる履歴ではなく、現在の文脈として参照される
- 以前の話題や約束事を自然に引き継げる
- 会話が毎回リセットされない

## ユースケース 2: 自発的な会話提案

ユーザーから入力がなくても、OtomeKairo が必要と判断したときに会話を始める。

期待する状態:

- 自発発話は単なる定期通知ではなく、文脈や関係性を踏まえたものになる
- 発話の必要有無は LLM が判断する
- 発話頻度の基準は OtomeKairo 側の設定として保持され、クライアントはその編集 UI として振る舞う

## ユースケース 3: 嗜好の学習

会話の中から、ユーザーの好き嫌いや反応傾向を学び、それが将来の応答に反映される。

期待する状態:

- 同じ質問に毎回初見のようには反応しない
- 好みや苦手を継続的な前提として扱える
- 学んだ内容によって提案や話題選びが変わる

## ユースケース 4: 他者を含む世界認識

OtomeKairo は、ユーザー以外の人や対象も認識し、会話の中で関係を維持する。

期待する状態:

- 自分、ユーザー、その他の人物や対象を区別できる
- 会話に登場した人物や出来事を継続的に扱える
- 世界の理解が、自分とユーザーの 2 者関係だけに閉じない

## ユースケース 5: 人格と記憶の切り替え

人格設定と記憶を、それぞれ独立に切り替えられる。

このユースケースの主目的は、同じ人格のまま複数の記憶を切り替えて挙動を比較しやすくすることである。

期待する状態:

- 同じ人格のまま、複数の記憶を切り替えて試せる
- 人格を変えても、記憶は自動では引き継がれない
- 記憶を変えても、人格は自動では変わらない
- どちらか一方だけを差し替えられる

## ユースケース 6: 将来の外界行動への拡張

将来的には、OtomeKairo が会話以外の行動も扱えるようにする。

この段階で期待する状態:

- 会話以外の行動も、会話と同じ意思決定の延長として扱える
- できることは、コード側が明示した能力の範囲で決まる
- 能力が増えても、設計の中心構造を壊さずに拡張できる
