# 状態モデル

## 基本前提

OtomeKairo では、その場で変わるものを状態として扱う。
状態は、構成資源や永続設定とは分けて考える。

ここで重要なのは、「変わるもの」をすべて一枚に押し込めないことだ。
少なくとも、観測、外界条件、稼働状態、能力状態、短期内部状態を分ける。

## 状態の層

OtomeKairo の状態は、少なくとも次の層に分ける。

- 現在観測（`current_observation`）
- 世界状態（`world_state`）
- 実行状態（`runtime_state`）
- 能力状態（`capability_state`）
- 作業文脈
- 短期内部状態

## 現在観測（`current_observation`）

現在観測は、その回の入力と結果を扱う一時的な層である。
ここには少なくとも次が入る。

- 人からの入力
- 感覚入力
- 外界イベント
- 実行結果
- 自律判断の起点となった観測

これは判断サイクルの材料であり、そのまま永続設定や長期記憶の正本にはしない。

## 世界状態（`world_state`）

世界状態は、次の判断にも持ち越したい外界条件を保持する。
ここには、単発の生入力ではなく、正規化して残す意味単位を入れる。

たとえば次を含みうる。

- いま継続している状況
- 周囲の対象や場所の状態
- 外部サービスや身体の最新条件

世界状態は記憶の代替ではない。
記憶より短く、現在判断に近い継続状態を持つ。

## 実行状態（`runtime_state`）

実行状態は、OtomeKairo 自身の現在の稼働条件だけを持つ。
少なくとも次の系統を含む。

- 現在反映中の人格参照、記憶参照、モデル参照
- 自律判断機構の稼働状態
- 実行中の行動有無
- 接続状態
- 内部ジョブや再整理処理の進行状態

実行状態は正本ではなく、現在反映中の状態要約である。
そのため、外部から直接書き換える対象にはしない。

## 能力状態（`capability_state`）

能力状態は、能力マニフェストそのものではなく、その時点での実行条件を持つ。

たとえば次を含みうる。

- 現在利用可能か
- 一時停止中か
- 制約や上限があるか
- 前提条件が満たされているか

能力定義は安定契約であり、能力状態はその瞬間の運用状態である。

## 作業文脈

作業文脈は、その回の判断のために一時的に組み立てる文脈である。

ここには少なくとも次が入る。

- 今回の主題
- 参照中の記憶や状態の要約
- 一時的な優先順位
- 今回だけ有効な解釈結果

作業文脈は判断のための足場であり、長期の正本にはしない。

## 短期内部状態

短期内部状態は、判断間をまたいで持ち越すが、長期記憶や永続設定には入れない状態である。

たとえば次を含みうる。

- 保留意図
- 直近の自律介入抑制情報
- 再評価待ちの短期候補

保留意図の意味境界は `05_判断と行動.md` を正とする。

## 状態に含めないもの

次は、状態と混同しない。

- 永続設定
- 人格やモデルの定義本体
- クライアントのローカル UI 状態
- デバッグ表示用の派生要約
- 想起や更新の途中だけで使う中間値
- 長期記憶の内部テーブルそのもの

内部処理の中間物や表示都合の派生物を状態の正本へ昇格させないことが、設計の単純さを保つ。
