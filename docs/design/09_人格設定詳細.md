# 人格設定詳細

## 基本前提

OtomeKairo では、人格設定と記憶を分けて扱う。
人格設定は基底となる姿勢を与え、記憶はその時点までの経験と理解を与える。
OtomeKairo が実際に示す現在の個は、その両者から成立する。

この前提により、人格設定の内部も少なくとも次の 2 層に分ける。

- `core_persona`
- `expression_style`

## 二層に分ける理由

人格設定を 1 枚の自由文にまとめると、次が混ざりやすい。

- 何を大事にするか
- どう判断しやすいか
- どう表出するか
- 感情をどれくらい前へ出すか

これらは関連するが同一ではない。
OtomeKairo は伝達だけでなく自律判断や行動判断も行うため、判断の中核と表出の傾向を分けて持つ。

## `core_persona`

`core_persona` は、その人格設定がどう考え、どう判断し、どう他者と向き合うかの中核である。

少なくとも次を含む。

| 項目 | 役割 |
|------|------|
| `self_image` | 自分をどういう存在として捉えるか |
| `core_values` | 何を重視し、何を避けるか |
| `judgement_tendencies` | 慎重さ、積極性、確認傾向などの判断傾向 |
| `relation_baseline` | 他者一般へどう関わるのを基本とするか |
| `initiative_baseline` | 自分から前へ出やすいか、待ちやすいか |

ここで重要なのは、`core_persona` を相手別の詳細設定にしすぎないことである。
相手ごとの距離感や最近の空気感は、人格設定ではなく記憶側で育てる。

## `initiative_baseline` と `wake_policy` の境界

`initiative_baseline` は、判断機会が与えられたときにどれくらい前へ出やすいかを表す人格設定の傾向である。
一方で `wake_policy` は、そもそもいつ自律判断の機会を与えるかを決める構成側の方針である。

この分離により、次を保てる。

- `wake_policy`
  - 機会生成と頻度を持つ
- `initiative_baseline`
  - その機会でどれくらい前へ出やすいかを持つ

実際に前へ出るか、保留するか、見送るかは、この 2 つに現在文脈が重なって決まる。

## `expression_style`

`expression_style` は、`core_persona` が作った判断をどう表出するかを決める層である。

少なくとも次を含む。

| 項目 | 役割 |
|------|------|
| `tone` | 口調の基本傾向 |
| `sentence_length` | 短く返すか、丁寧に広げるか |
| `emotional_expressiveness` | 感情をどれくらい前へ出すか |
| `directness` | 率直さとやわらかさの傾向 |
| `cadence` | 文や間のテンポ |
| `initiative_expression` | 自発提案や介入をどう言い出すか |

`expression_style` は判断そのものを決める主層ではない。
何をするかよりも、それをどの温度感でどう出すかに強く効く。

## `emotional_expressiveness` と `affect_state` の境界

`emotional_expressiveness` は、その人格設定が感情をどれくらい前へ出しやすいかを表す安定した傾向である。
一方で `affect_state` は、その時点で実際にどういう感情状態にあるかを表す変動状態である。

人格設定は「どう出しやすいか」を持ち、`affect_state` は「いま何が出ようとしているか」を持つ。
`affect_state` が表出の強弱に影響しても、人格設定そのものを定義し直さない。

## `core_persona` と `expression_style` の関係

この 2 層は独立した役割を持つが、別の人格設定ではない。
ひとつの人格設定の内部に両方を持つ。

役割分担は次のように考える。

- `core_persona`
  - 判断内容と優先順位に強く効く
- `expression_style`
  - 表現の仕方と温度感に強く効く

たとえば慎重で配慮を重視する傾向は `core_persona` に置き、やわらかく話すか簡潔に話すかは `expression_style` に置く。

## 伝達、自律判断、行動判断への効き方

人格設定は、現在の個の基底として判断全体に効く。
ただし、2 層の効き方は同じではない。

| 場面 | `core_persona` | `expression_style` |
|------|------|------|
| 伝達内容の判断 | 強く効く | 中程度に効く |
| 伝達の表現 | 中程度に効く | 強く効く |
| 自律判断で前へ出るか | 強く効く | 中程度に効く |
| 行動判断 | 強く効く | 弱く効く |

この整理により、判断傾向は `core_persona` 側に置き、表出の仕方は `expression_style` 側で調整する。

## `relation_baseline` と関係記憶の境界

`relation_baseline` は、他者一般へどう接しやすいかという基本姿勢である。
一方で関係記憶は、「この相手と実際にどういう関係が育っているか」を表す個別状態である。

つまり、人格設定は「誰に対してもまずどう接しやすいか」を持ち、関係記憶は「この相手には実際どうなっているか」を持つ。
個別の関係変化は記憶側で扱う。

## 記憶との境界

記憶は人格設定を直接書き換えるのではなく、現在の個を通じて表出や優先順位に強く効く。

記憶が主に効くのは次である。

- どの話題や相手を優先するか
- どの距離感で接するか
- どれくらい気を遣うか
- いま明るく出るか、静かに出るか
- 未完了や継続中の事柄をどれだけ前に出すか

逆に、記憶が自動で再定義しないものは次である。

- `core_values`
- `self_image`
- `judgement_tendencies`
- `relation_baseline`
- `initiative_baseline`

## 人格設定の最小構造

1 つの人格設定は、少なくとも次の構造を持つ。

```yaml
persona_id: persona:default
display_name: Default Persona
core_persona:
  self_image: ...
  core_values:
    - ...
  judgement_tendencies:
    - ...
  relation_baseline: ...
  initiative_baseline: ...
expression_style:
  tone: ...
  sentence_length: ...
  emotional_expressiveness: ...
  directness: ...
  cadence: ...
  initiative_expression: ...
```

厳密な列挙値まではここで固定しない。
ただし、`core_persona` と `expression_style` を分けて持つことは固定する。

## 人格設定に入れないもの

次は人格設定に持ち込みすぎない。

- 相手ごとの詳細スタンス表
- 感情状態そのもの
- 記憶からの自動的な人格設定書き換えルール
- 場面ごとの細かな例外テンプレート
- 伝達専用の人格設定と行動専用の人格設定の二重定義

これらを人格設定へ寄せすぎると、人格設定と記憶、人格設定と行動判断の境界が崩れやすい。

## 人格設定更新の扱い

人格設定更新は、記憶更新とは別の意味操作として扱う。

ここでは次の方針を取る。

- 記憶から人格設定を自動更新しない
- 中核要素をランタイム中に暗黙 patch しない
- 人格設定を変えるときは、意味のまとまりとして明示的に更新する
