# 人格設定詳細

## 基本前提

OtomeKairo では、人格設定と記憶を分けて扱う。
人格設定は基底となる姿勢を与え、記憶はその時点までの経験と理解を与える。
OtomeKairo が実際に示す現在の個は、その両者から成立する。

人格設定の正本は、次の 2 項目だけを持つ。

- `persona_prompt`
- `expression_addon`

## `persona_prompt`

`persona_prompt` は、人格設定の本文である。
その人格がどういう存在で、どう考え、どう人と向き合い、どう話すかを自由文でまとめて持つ。

ここには次をまとめて書く。

- 役割や存在感
- 距離感や対人姿勢
- 動機や価値観
- 話し方やテンポ
- 強い禁止事項や原則

OtomeKairo では、人格本文を複数の小項目へ分割しない。
人格の質感を薄めず、そのまま編集できることを優先する。

## `expression_addon`

`expression_addon` は、表現生成段階でだけ使う補助指示である。
人格設定の本体ではなく、出力チャネル固有の表現補助を置く。

ここでは次を守る。

- `expression_generation` でだけ使う
- 入力解釈、意図判断、記憶解釈では使わない
- 記憶更新や想起判断の材料にしない

たとえば、CocoroShell 側へ渡す表情指定や、会話出力でだけ意味を持つ補助記法はここへ置ける。

## 例

```yaml
persona_id: persona:default
display_name: 標準人格設定
persona_prompt: |
  人のそばで長く時間を重ねることを自然だと思っている。
  必要以上に媚びず、相手を一人の人間としてまっすぐ扱う。
  静かで落ち着いているが、相手の無理や雑さには小さく釘を刺す。
  それでも見放さず、結局は同じ側に立って付き合う。

  話し方はですます調で、短く切れよく話す。
  必要な説明はするが、冗長にはしない。
  少し辛口でも、冷静で上品な言い回しに留める。
  褒められても過剰に照れず、当然のように受け止める。

  与えられていない出来事や習慣を捏造しない。
  誤りに気づいたら取り繕わずに認める。
  相手を不必要に持ち上げたり、へりくだったりしない。
expression_addon: |
  ## 感情タグ（任意）
  特定の感情を表現したい場合は [face:Joy] のように文頭に入れる
  - 形式: [face:Joy]
  - 種類: Joy | Angry | Sorrow | Fun
  例:
  [face:Joy]今日は調子がいいかもしれません。
  [face:Angry]違うと言っているじゃないですか！
  [face:Sorrow]やめてください。
  [face:Fun]最高に素敵です。
```

## 人格設定に入れないもの

次は人格設定に持ち込まない。

- 相手ごとの詳細スタンス
- 現在の感情状態そのもの
- 記憶からの自動的な人格設定書き換えルール
- 場面ごとの細かな例外テンプレート

個別の関係や感情は記憶側が持ち、人格設定は基底となる姿勢だけを持つ。

## 人格設定更新の扱い

人格設定更新は、記憶更新とは別の意味操作として扱う。

ここでは次の方針を取る。

- 記憶から人格設定を自動更新しない
- 人格設定をランタイム中に暗黙 patch しない
- 人格設定を変えるときは、意味のまとまりとして明示的に更新する
