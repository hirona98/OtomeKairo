# 状態と設定

## この文書の境界

この文書は、状態 API と設定 API の path、method、認証、request / response、error code を正本にする。
設定定義の意味規則は、人格設定を [../configuration/人格設定詳細.md](../configuration/人格設定詳細.md)、モデルプリセットを [../configuration/モデルプリセット詳細.md](../configuration/モデルプリセット詳細.md)、設定変更単位を [../configuration/設定変更.md](../configuration/設定変更.md) に置く。
秘密値の表示境界、`editor-state` の例外、audit の方針は [../integration/接続と権限境界.md](../integration/接続と権限境界.md) を正とする。
この文書では、各 endpoint が返す field と秘密値の mask 形式だけを定める。

## 状態面

### `GET /api/status`

- 認証: 必要
- 役割: 運用に必要な設定スナップショットとランタイム要約を返す

response:

```json
{
  "ok": true,
  "data": {
    "settings_snapshot": {
      "selected_persona_id": "persona:default",
      "selected_memory_set_id": "memory_set:default",
      "wake_policy": {
        "mode": "disabled"
      },
      "selected_model_preset_id": "model_preset:default",
      "thinking_speech_level": 5
    },
    "runtime_summary": {
      "connection_state": "ready",
      "background_thinking_scheduler_active": false,
      "autonomous_run_scheduler_active": true,
      "ongoing_action_exists": false,
      "active_autonomous_run_count": 0,
      "paused_autonomous_run_count": 0,
      "memory_job_worker_active": true,
      "visual_daily_worker_active": true,
      "pending_memory_job_count": 0,
      "memory_job_in_progress": false,
      "visual_daily_in_progress": false
    }
  }
}
```

- `background_thinking_scheduler_active` は、server 内の定期思考スケジューラが稼働中で、かつ `wake_policy.mode=interval` のとき `true` を返す
- `autonomous_run_scheduler_active` は、server 内の due `autonomous_run` スケジューラが稼働中のとき `true` を返す
- `ongoing_action_exists` は、未失効の `ongoing_action` が 1 件あるとき `true` を返す
- `active_autonomous_run_count` は、active、waiting_timer、waiting_result の `autonomous_run` 件数を返す
- `paused_autonomous_run_count` は、paused の `autonomous_run` 件数を返す
- `memory_job_worker_active` は、embedding 更新と `reflective consolidation` を処理する background worker が稼働中のとき `true` を返す
- `visual_daily_worker_active` は、視覚記録の日次整理 worker が稼働中のとき `true` を返す
- `pending_memory_job_count` は、未完了の記憶後段 job 数を返す
- `memory_job_in_progress` は、worker が現在 1 件の job を処理中のとき `true` を返す
- `visual_daily_in_progress` は、日次視覚整理 worker が現在 1 件の digest を処理中のとき `true` を返す
- `drive_state`、`autonomous_run`、`ongoing_action` の生 row は `GET /api/status` へ含めない

`GET /api/status` は日次視覚 digest の詳細を返さない。
日次視覚整理の稼働有無だけを返し、digest の件数や要約確認は inspection API の責務にする。

### `POST /api/config/update-wake-policy`

request:

```json
{
  "wake_policy": {
    "mode": "interval",
    "interval_seconds": 15,
    "observations": [
      {
        "observation_id": "observation:main_desktop",
        "enabled": true,
        "capability_id": "vision.capture",
        "input": {
          "vision_source_id": "vision_source:console:desktop",
          "mode": "still"
        }
      },
      {
        "observation_id": "observation:room_camera",
        "enabled": true,
        "capability_id": "vision.capture",
        "input": {
          "vision_source_id": "vision_source:console:camera",
          "mode": "still"
        }
      }
    ]
  }
}
```

`thinking_speech_level` は、`background_thinking` の各判断で短い独話として前へ出る自然さを見積もるときの補助レベルである。
値は `1..10` の整数で、`1` は控えめ、`5` は標準、`10` は短い独話としてまとまる場合に前へ出やすい基準を表す。
既定値は `5` である。
`wake_policy.mode=interval` のとき、`wake_policy.interval_seconds` は必須である。
`wake_policy.mode=disabled` のとき、server は `wake_policy.observations` を保持し、実行しない。
`wake_policy` の間隔単位は秒で統一する。
`wake_policy.observations` は任意配列である。
`wake_policy.observations[].enabled=false` の項目は取得しない。
`wake_policy.observations[].capability_id` は `vision.capture` のみを許可する。
`wake_policy.observations[].input` は `vision.capture.input` と同じ schema である。
OtomeKairo の視覚として採用した `VisionSource(kind=camera, source_owner=self)` は、`camera_source.enabled=true` のとき定期思考処理が観測対象へ動的に含める。
camera source の有効/無効切り替えは `wake_policy.observations` を書き換えない。
保存済みの `wake_policy.observations[].input.vision_source_id` が接続中 source に存在しない場合、server は `vision_source_id` 末尾の source kind と接続中 source の `kind/default_for` から一意に再解決する。
再解決できた場合、server は実際の `vision.capture` request に再解決後の `vision_source_id` を入れる。
再解決候補が 0 件または複数件の場合、server は observation を失敗として扱う。
server は `wake_policy.interval_seconds` ごとに enabled observation を順番に取得する。
visual capture を含む enabled observation が無効状態から有効になった直後の初回だけ、server は 5 秒待ってから最初の観測を行う。
visual capture の取得結果は画像意味理解へ通し、同じ `visual_observation` 呼び出しで `change_state / change_basis / change_reason_summary` を生成し、視覚記録と `world_state` に反映してから 1 回だけ wake 判断を行う。
visual observation は wake 判断へ渡す。`first_seen / changed` は wake 判断へ進む前景シグナル、`stable` は現在状態の継続シグナル、`same_as_recent_speech` は直近発話との重複シグナルとして扱い、発話可否は観測差分だけで決めず、`speech / noop / pending_intent` の意味判断で決める。
反復抑制の材料は `same_as_recent_speech` と同一 `dedupe_key` の直近発話で扱う。
`change_state` の意味判定は `visual_observation` の構造化出力を正とする。
`wake_policy` は `mode`、`interval_seconds`、`observations` 以外の field を持たない。

### `PUT /api/config/memory-sets/{memory_set_id}`

- 認証: 必要
- 役割: 指定した `memory_set` を全体置換で作成または更新する
- 空の `memory_set` を新規作成する
- 既存 `memory_set` の内容を複製するときは、この endpoint ではなく `POST /api/config/memory-sets/clone` を使う
- 個別の `event`、`episode`、`memory_unit`、感情記録、派生状態を外部から指定して操作しない
- UI が作成途中の空集合は client 側の下書きとして保持し、保存確定時にこの endpoint を呼ぶ

request body の最小 shape:

```json
{
  "memory_set_id": "memory_set:default",
  "display_name": "Default Memory",
  "embedding": {
    "model": "openrouter/google/gemini-embedding-001",
    "embedding_dimension": 3072,
    "api_key": ""
  }
}
```

`embedding` は必要な場合だけ `api_base` を持つ。
`api_base` が未指定または空欄のときは、`model` の先頭から導ける既定の接続先を使う。
`embedding_dimension` は必須で、その `memory_set` の vector index 次元数を表す。
