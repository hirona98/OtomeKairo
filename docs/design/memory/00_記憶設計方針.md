# 記憶設計方針

<!-- Block: Role -->
## この文書の役割

この文書は、記憶設計ファミリーの入口である。

ここで残すのは次だけに絞る。

- 記憶を人格形成の基盤として扱う理由
- 記憶設計ファミリーの読む順
- どの文書が正本で、どの文書が補助か

記憶の厳密な契約そのものは、`01_記憶システム概要.md` から `04_記憶更新と再整理.md` を正とする。

<!-- Block: Why -->
## なぜ記憶を重く扱うか

OtomeKairo では、人格と記憶を操作上は独立に扱う。
ただし、実際の応答や自発判断は人格設定だけで決まらない。

現行設計では、次を前提にする。

- 人格設定は基本姿勢を与える
- 記憶は自己像、他者理解、関係、未完了、感情傾向を育てる
- 応答や自発判断は、その時点の人格設定と記憶から組み上がる現在文脈で決まる

そのため、記憶は検索補助ではなく、人格形成の基盤として設計する。

<!-- Block: FixedDecisions -->
## 現時点で固定している判断

現行設計では、少なくとも次を固定している。

- 永続記憶は `events / episode_digests / memory_units / affect_state` の 4 層に分ける
- `memory_units` は自然文だけでなく半構造化項目を持つ
- 想起は役割別 `RecallPack` として返す
- 更新は `turn consolidation` と `reflective consolidation` の 2 粒度で扱う
- 根拠、履歴、状態遷移を最初から持つ

<!-- Block: NotAdopted -->
## 意図的に採らないもの

現時点では、次は採らない。

- ベクトル検索だけで記憶を成立させること
- なんでも 1 本の自由文 summary に潰すこと
- 生ログを捨てて派生記憶だけ残すこと
- 1 ターンごとに自己像全体を書き直すこと
- 最初から大規模知識グラフを前提にすること
