# 記憶管理と削除

## 目的

この文書は、個別記憶行に対する外部操作と管理操作を採用しない境界を定める。

OtomeKairo の記憶は、利用者や管理者が直接編集するプロフィールデータベースではない。
個別の記憶内容を変更できる主体は OtomeKairo 自身の記憶更新処理だけである。
認証済みの通常 API でも、個別の `event`、`episode`、`memory_unit`、感情記録、派生状態を直接変更しない。

## 基本方針

個別記憶行に対して、管理上の削除、伏せ字化、保持期限処理、export を提供しない。
`memory_set` 全体の複製と削除は、記憶行編集ではなく設定定義操作として扱う。

採用する方針は次である。

- 管理者権限を設けない
- 認証済み主体にも個別記憶行の直接編集権限を与えない
- 個別記憶行の内容変更は、通常入力、観測、実行結果、内部再整理を OtomeKairo が解釈した結果としてだけ行う
- 利用者からの訂正、否定、撤回要求は通常入力として扱い、物理削除命令として扱わない
- inspection は読み取り面であり、記憶編集面にしない
- 設定 API は設定定義と選択値を扱い、記憶本体の行編集を扱わない

## 許可する変更経路

記憶に変化を起こせる経路は次に限る。

| 経路 | 変更主体 | 役割 |
|------|----------|------|
| `turn consolidation` | OtomeKairo | 入力と結果を `events / episodes / memory_units` へ反映する |
| `reflective consolidation` | OtomeKairo | 蓄積した記憶から再整理、要約、派生状態更新を行う |
| 感情更新 | OtomeKairo | `episode_affect / mood_state / affect_state` を更新する |
| 派生状態更新 | OtomeKairo | `drive_state / pending_intent / world_state` などを再構築または失効させる |
| vector index 同期 | OtomeKairo | 記憶本体から検索用 index を再生成する |

これらは、外部利用者が操作名を直接指定して実行する管理 API ではない。
OtomeKairo の判断サイクルまたは内部 worker が、正本記憶と設計済み validator に従って実行する。

## 採用しない操作

次の操作は提供しない。

| 操作 | 方針 |
|------|------|
| `export_memory_set` | 提供しない |
| `redact_event_payload` | 提供しない |
| `redact_trace_payload` | 提供しない |
| `delete_memory_scope` | 提供しない |
| `apply_retention_policy` | 提供しない |
| 個別行 edit | 提供しない |

`memory_set` 全体の削除は設定 API の `DELETE /api/config/memory-sets/{memory_set_id}` で扱う。
記憶集合全体の export、初期化、保持期限による整理は行わない。
raw payload や trace の後追い伏せ字化も行わない。
秘密値や不要な payload を残さない責務は、入力、記録、ログ出力の前段で満たす。

## 訂正と撤回要求

利用者が「それは違う」「その話は扱わないでほしい」と伝えた場合、その発話は通常入力として扱う。
OtomeKairo は既存の根拠と照合し、必要なら `supersede`、`revoke`、`dormant` などの記憶更新として反映する。

これは物理削除ではない。
過去に起きた入力や更新履歴を、外部要求によって消す処理にしない。

## API 境界

個別記憶行の記憶管理 API は追加しない。
通常 API、設定 API、inspection API、stream API のいずれにも、個別記憶行の削除、伏せ字化、保持期限処理、export を入れない。

`memory_set` は現在選択する記憶集合の単位であり、設定 API で集合単位に作成、複製、削除する。
`memory_set` の複製と削除は集合全体を対象にし、内部の個別行を外部から選んで操作する面にしない。
空の `memory_set` 作成と `memory_set` 複製は分ける。
`memory_set` の初期化と export は提供しない。

## やらないこと

次は採らない。

- 管理者権限を追加すること
- 認証済み API から記憶行を直接編集すること
- 個別記憶行を直接操作すること
- 記憶本体を export すること
- 記憶本体や trace を後追いで伏せ字化すること
- 保持期限で記憶本体や raw payload を削除すること
- `memory_set` 全体を初期化すること
- 利用者の通常入力を物理削除命令として扱うこと
