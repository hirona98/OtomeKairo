# 記憶システム概要

## 基本前提

OtomeKairo では、記憶を単なる検索用ストレージとして扱わない。
記憶は、自己像、他者理解、関係、未完了、感情傾向を育て、その時点の人格文脈を構成する中核として扱う。

この前提により、次が成立する。

- 記憶は会話品質の補助ではなく、人格形成の基盤である
- 会話と行動は、同じ記憶文脈から分岐する
- 生ログ、エピソード、継続理解、感情状態を分けて扱う

## 記憶システムの基本構造

OtomeKairo の記憶システムは、次の 4 層で構成する。

1. `events`
2. `episode_digests`
3. `memory_units`
4. `affect_state`

各層の役割は次のとおりである。

- `events`
  - 観測された生ログの正本
- `episode_digests`
  - 想起しやすい出来事単位
- `memory_units`
  - 継続理解の最小単位
- `affect_state`
  - 瞬間感情と背景感情

このうち、人格形成の中核になるのは `memory_units` である。
ただし、`memory_units` だけでは足りない。`episode_digests` と `events` を根拠層として参照する。

## 記憶が使われる循環

記憶は、次の循環の中で使われる。

1. 現在の観測を受け取る
2. `RecallHint` を作る
3. `RecallPack` を組む
4. MVP では内部結果 `reply / noop / future_act` を判断する
5. 結果を `events` に残す
6. `turn consolidation` で記憶を更新する
7. トリガー条件を満たしたときに `reflective consolidation` で再整理する

重要なのは、「返答時に思い出すこと」と「返答後に育てること」を分けることである。
将来の外界行動拡張では、この内部判断から `act` 分岐を外向き結果として持つ。
ただし、MVP の正本は `reply / noop / future_act` であり、外向き API は `reply / noop / internal_failure` に留める。
