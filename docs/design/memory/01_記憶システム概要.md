# 記憶システム概要

## 基本前提

OtomeKairo では、記憶を単なる検索用ストレージとして扱わない。
記憶は、自己像、他者理解、関係、未完了、感情傾向を育て、その時点の現在の個の文脈を構成する中核として扱う。

この前提により、次が成立する。

- 記憶は会話品質の補助ではなく、現在の個を構成する基盤である
- 会話と行動は、同じ記憶文脈から分岐する
- 生ログ、経験のまとまり、継続理解、感情状態を分けて扱う

## 記憶システムの基本構造

OtomeKairo の記憶は、次の 4 層で構成する。

1. `events`
2. `episodes`
3. `memory_units`
4. `affect_state`

各層の役割は次のとおりである。

- `events`
  - 観測、返答、判断、実行結果の生ログを残す根拠層
- `episodes`
  - その場で何が起き、何が残ったかを束ねる経験単位
- `memory_units`
  - 経験から育った継続理解の単位
- `affect_state`
  - 瞬間感情と背景感情

現在の個を支えるのはこの 4 層の全体である。
その中で、`memory_units` は継続理解の中心であり、`episodes` は経験連続性の中心である。
`events` は根拠層として残り、`affect_state` は判断の前景を変える。

## 補助構造

次は記憶の正本層ではなく、想起、監査、索引を支える補助構造として扱う。

- `memory_links`
  - 理解同士の関係
- `revisions`
  - `memory_units` の更新履歴
- `retrieval_runs`
  - 想起時の選定記録
- `vector_index_entries`
  - 連想レーン用の補助索引

これらは重要だが、現在の個を直接構成する記憶層ではない。

## 記憶が使われる循環

記憶は、次の循環の中で使われる。

1. 現在の観測を受け取る
2. `RecallHint` を作る
3. `RecallPack` を組む
4. 判断系が結果種別を選ぶ
5. 結果を `events` に残す
6. `turn consolidation` で `episodes` と `memory_units` を更新する
7. トリガー条件を満たしたときに `reflective consolidation` で再整理する

重要なのは、「判断時に思い出すこと」と「結果を受けて育てること」を分けることである。
想起は現在の個の文脈を作り、更新は次回以降の現在の個を育てる。
