# 想起と判断フロー図

## 読み方

この文書は、次の順で読むと分かりやすい。

1. 全体ライフサイクル
2. 同期想起の詳細
3. 判断分岐の詳細
4. `RecallPack` への振り分け
5. 結果後の更新

特に重要なのは、OtomeKairo の判断が 1 本線ではなく、次の 4 層に分かれていることである。

- 同期想起
  - 判断直前に必要分だけ読む
- 判断分岐
  - 想起後に結果種別を決める
- ターン更新
  - 結果後に 1 ターン単位で記憶を育てる
- 反省整理
  - 数ターンまたは一定時間ごとに長期変化を再評価する

## 全体ライフサイクル

```mermaid
sequenceDiagram
    autonumber
    actor Obs as 観測源
    participant RT as Runtime
    participant RH as RecallHintBuilder
    participant SR as 構造レーン
    participant VR as 連想レーン
    participant RP as RecallPackBuilder
    participant DJ as Decision
    participant TC as Turn Consolidation
    participant RC as Reflective Consolidation
    participant EV as events
    participant EP as episodes
    participant MU as memory_units
    participant AF as affect_state
    participant VX as vector_index

    Obs->>RT: 観測入力
    RT->>RH: 現在観測 + 直近会話 + 現在時刻
    RH-->>RT: RecallHint

    par 構造レーン
        RT->>SR: RecallHint
        SR->>MU: scope / memory_type / status で取得
        SR-->>RT: hard candidates
    and 連想レーン
        RT->>VR: RecallHint
        VR->>MU: vector top-k
        VR->>EP: vector top-k
        VR-->>RT: associative candidates
    end

    RT->>RP: hard + associative candidates
    RP->>RP: section 再配置
    opt 細部確認が必要
        RP->>EV: selected linked_event_ids を限定ロード
        EV-->>RP: raw event slices
        RP->>RP: event_evidence へ圧縮
    end
    RP-->>RT: RecallPack

    RT->>DJ: 人格設定 + 記憶文脈 + 現在観測 + 状態群
    DJ-->>RT: respond / act / hold / noop

    alt 応答
        RT->>TC: 観測 / 応答 / 判断結果
    else 行動
        RT->>TC: 観測 / 行動結果
    else 保留
        RT->>TC: 観測 / 保留判断 / 保留状態更新結果
    else 何もしない
        RT->>TC: 観測 / noop 判断
    end

    TC->>EV: events 保存
    TC->>EP: 1判断サイクル1episode 生成
    TC->>MU: create / reinforce / refine / supersede / revoke / dormant
    TC->>AF: 感情更新
    TC->>VX: 埋め込み更新要求

    opt トリガー到達
        RT->>RC: 8ターン / 24h / 訂正 / 関係変化
        RC->>EP: 複数 episode を横断
        RC->>MU: summary 生成 / confirmed 見直し / dormant 化
        RC->>VX: 再索引
    end
```

## 同期想起の詳細

```mermaid
flowchart TD
    A[現在観測] --> B[直近3分以内・最大4-6発話]
    A --> C[RecallHint 生成]
    B --> C
    C --> D[primary_intent / secondary_intents / time_reference / focus_scopes]

    D --> E1
    D --> E2

    subgraph SL[構造レーン]
        direction TB
        E1[hard requirement 抽出]
        F1[active_commitments]
        F2[relationship_model]
        F3[user_model]
        F4[self_model]
        F5[active_topics]
        G1[厳密候補集合]
        E1 --> F1
        E1 --> F2
        E1 --> F3
        E1 --> F4
        E1 --> F5
        F1 --> G1
        F2 --> G1
        F3 --> G1
        F4 --> G1
        F5 --> G1
    end

    subgraph VL[連想レーン]
        direction TB
        E2[問い合わせ方向の決定]
        H1[現在観測を主クエリ化]
        H2[mentioned_entities / topics で補強]
        H3[primary_intent / time_reference で重み調整]
        H4[memory_units vector top-k]
        H5[episodes vector top-k]
        H6[連想候補集合]
        E2 --> H1
        E2 --> H2
        E2 --> H3
        H1 --> H4
        H1 --> H5
        H2 --> H4
        H2 --> H5
        H3 --> H4
        H3 --> H5
        H4 --> H6
        H5 --> H6
    end

    G1 --> I[section ごとに再配置]
    H6 --> I
    I --> J[構造候補を優先]
    J --> K[vector-only 候補は弱く扱う]
    K --> L[選ばれた memory_units に evidence 付与]
    L --> X{細部確認}
    X -- no --> M[conflicts 抽出]
    X -- yes --> Y[selected linked_event_ids を限定ロード]
    Y --> Z[event_evidence を短文化]
    Z --> M
    M --> N[全体上限 8-14 件]
    N --> O[RecallPack 確定]
```

## 判断分岐の詳細

```mermaid
flowchart TD
    A[人格設定] --> E[Decision]
    B[RecallPack] --> E
    C[現在観測] --> E
    D[状態群] --> E

    E --> G{分岐}

    G -->|respond| H[応答を生成]
    G -->|act| I[行動を実行]
    G -->|hold| J[保留状態を更新]
    G -->|noop| K[今回は何もしない]
```

```mermaid
flowchart LR
```
