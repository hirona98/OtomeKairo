# 想起と判断ユースケース

## この文書が扱う対象

この文書では、特に次を対象とする。

- `RecallHint` が自然な向き付けを作れる範囲
- 構造レーンと連想レーンの組み合わせで必要な記憶を直前に集められる範囲
- `RecallPack` の section 分離で、現在の個の文脈と話題理解を保てる範囲
- この設計の責務外がどこにあるか

この文書は、実装性能やモデル選定ではなく、想起設計の適用範囲を整理する。

## 適用範囲の分類

各パターンは次の 3 区分で整理する。

| 分類 | 意味 |
|------|------|
| `標準で扱う` | 標準経路で安定して処理できる |
| `条件がある` | 条件が揃えば処理できるが、誤想起や取りこぼしの余地がある |
| `責務外` | この設計の責務外か、別の機構がないと不安定になる |

`標準で扱う` とする条件は次である。

- `RecallHint` が現在入力と直近文脈だけで十分な向き付けを作れる
- 構造レーンか連想レーンのどちらか、または両方で必要候補を取れる
- `RecallPack` に入った情報だけで、返答や判断が断定しすぎず自然に組み立てられる

## 標準で扱うパターン

### パターン A1: 未完了の約束確認

| 項目 | 内容 |
|------|------|
| 分類 | `標準で扱う` |
| 入力例 | 「この前の続き、また今度話すって言ってたよね」 |
| 主に効く section | `active_commitments`, `relationship_model`, `episodic_evidence` |
| 理由 | `commitment` は構造レーンで常に優先され、ベクトルに依存せず拾えるため |

### パターン A2: 安定した嗜好の再利用

| 項目 | 内容 |
|------|------|
| 分類 | `標準で扱う` |
| 入力例 | 「辛いものの話してたけど、今度また食べに行きたい？」 |
| 主に効く section | `user_model`, `active_topics`, `episodic_evidence` |
| 理由 | 安定嗜好は `memory_units.preference` として残しやすく、話題一致で構造レーンから拾えるため |

### パターン A3: 最近の体調や悩みの継続相談

| 項目 | 内容 |
|------|------|
| 分類 | `標準で扱う` |
| 入力例 | 「最近また眠れてなくて、どうしたらいいかな」 |
| 主に効く section | `user_model`, `active_topics`, `relationship_model`, `episodic_evidence` |
| 理由 | 近況確認と相談は `user` / `topic` 軸で構造的に集めやすく、連想レーンも似た相談文脈を補助しやすいため |

### パターン A4: 関係そのものの確認

| 項目 | 内容 |
|------|------|
| 分類 | `標準で扱う` |
| 入力例 | 「最近ちょっと距離ある感じする？」 |
| 主に効く section | `relationship_model`, `episodic_evidence`, `conflicts` |
| 理由 | `relationship_model` を独立 section にした効果が最も出るパターンであり、返答の断定度も `conflicts` で抑制できるため |

### パターン A5: 日をまたいだ明示的な話題継続

| 項目 | 内容 |
|------|------|
| 分類 | `標準で扱う` |
| 入力例 | 「昨日話してた仕事の件だけど」 |
| 主に効く section | `active_topics`, `episodic_evidence`, `user_model` |
| 理由 | 過去参照が明示されていれば、`active_topics` と `episodes` で自然につなぎやすいため |

### パターン A6: 明示された第三者を含む会話継続

| 項目 | 内容 |
|------|------|
| 分類 | `標準で扱う` |
| 入力例 | 「田中さんの件、その後どう思ってる？」 |
| 主に効く section | `user_model`, `active_topics`, `episodic_evidence` |
| 理由 | 対象が明示されていれば、`mentioned_entities` から `scope_type=entity` の構造候補を取り、連想レーン補強と話題スコープで候補を狭められるため |

## 条件があるパターン

### パターン B1: 回想と相談が混ざった入力

| 項目 | 内容 |
|------|------|
| 分類 | `条件がある` |
| 入力例 | 「前の続きなんだけど、また相談してもいい？」 |
| 成立理由 | `RecallHint` が `interaction_mode`、`primary_recall_focus`、`secondary_recall_focuses`、`risk_flags` を返せるので、最低限の方向付けと危うさの共有はできる |
| 制約 | `primary_recall_focus` は 1 つだけなので、回想と相談の完全な両立最適化は弱い |

### パターン B2: 参照語が曖昧で、直近窓の外に文脈がある

| 項目 | 内容 |
|------|------|
| 分類 | `条件がある` |
| 入力例 | 「あの件、まだ気になってる？」 |
| 成立理由 | 直近窓に対象が残っていれば自然に補完できる |
| 制約 | 対象が直近窓の外にあると、`RecallHint` は長期記憶を読まないため、`risk_flags` で曖昧さは渡せても入口で対象特定に失敗しやすい |

### パターン B3: 明示手がかりの薄い昔話の回想

| 項目 | 内容 |
|------|------|
| 分類 | `条件がある` |
| 入力例 | 「なんか前にもこんな感じのことあったよね」 |
| 成立理由 | 連想レーンが `episodes` をベクトル検索して、似た空気感の経験を拾える |
| 制約 | 手がかりが少なすぎると、ベクトル hit の質に強く依存し、外しやすい |

### パターン B4: 最近の矛盾を含む自己申告

| 項目 | 内容 |
|------|------|
| 分類 | `条件がある` |
| 入力例 | 「前は辛いの好きって言ったけど、今はちょっと苦手かも」 |
| 成立理由 | `conflicts` により断定を避け、確認寄りの返答に落とせる |
| 制約 | どちらを有効な理解として採るかは、更新規則と再整理の質に依存する |

### パターン B5: 小さなやり取りの積み重ねで生じる関係変化

| 項目 | 内容 |
|------|------|
| 分類 | `条件がある` |
| 入力例 | 「最近なんとなく話しにくい気がする」 |
| 成立理由 | `relationship_model` と `episodic_evidence` があれば、確認寄りの返答はできる |
| 制約 | 微小な変化が複数ターンに分散している場合、`reflective consolidation` の質に強く依存する |

### パターン B6: 場所や出来事を鍵にした「何を話していたか」確認

| 項目 | 内容 |
|------|------|
| 分類 | `条件がある` |
| 入力例 | 「水族館に行ったとき、何のこと話してたんだっけ？」 |
| 成立理由 | `episodes` に場所や出来事のアンカーが残っていれば、その `linked_event_ids` から例外的に細部確認できる |
| 制約 | 場所アンカーが `episode` に残っていない場合や、1 `episode` に複数話題が圧縮されている場合は粗くなる |

## 責務外のパターン

### パターン C1: `events` 全文の広域再読が前提の質問

| 項目 | 内容 |
|------|------|
| 分類 | `責務外` |
| 例 | 長い会話全体の逐語再現、全文検索前提の確認 |
| 理由 | この設計は `events` 全文再読を標準経路にしないため |

### パターン C2: 複数の独立意図を同時に最適化する入力

| 項目 | 内容 |
|------|------|
| 分類 | `責務外` |
| 例 | 相談、確認、依頼を同時に高品質で処理する入力 |
| 理由 | `primary_recall_focus` を 1 本に絞る設計だから |

### パターン C3: 多人数の関係図や長い因果連鎖の推論

| 項目 | 内容 |
|------|------|
| 分類 | `責務外` |
| 例 | 複数人物の複雑な立場変化を一度に整理する質問 |
| 理由 | この設計は広い関係グラフ探索や多段推論を主責務にしていないため |

### パターン C4: 長期記憶に存在しない暗黙情報の補完

| 項目 | 内容 |
|------|------|
| 分類 | `責務外` |
| 例 | まだ残っていない含意や雰囲気だけから詳細を断定する質問 |
| 理由 | 想起系は、保存された記憶を優先し、存在しない情報を補完して作らないため |
