# 時刻モデル

## 基本方針

OtomeKairo の時刻は、OtomeKairo が生活するローカル時刻を正本にする。
UTC を判断文脈の正本にしない。

保存、API、inspection で扱う機械用 timestamp は、ローカルタイムゾーンに属する offset 付き ISO 8601 とする。

```text
2026-04-25T09:00:00+09:00
```

timezone 表記のない timestamp は使わない。

```text
2026-04-25T09:00:00
```

上の形はローカル時刻として扱わない。

## ローカルタイムゾーン

OtomeKairo 本体は、自分の生活タイムゾーンを 1 つ持つ。
たとえば `Asia/Tokyo` を生活タイムゾーンとして扱う。

生活タイムゾーンは、未来の予定、繰り返し wake、外部 timestamp の変換、監査表示の基準に使う。
各 row に timezone id を重複保存しない。
保存する timestamp には、その時点の offset を含める。

## LLM へ渡す時刻

LLM へ渡す時刻は、機械用 ISO timestamp を主表現にしない。
同じ時刻を生活文脈向けに整形した自然文として渡す。

```text
現在時刻: 2026年4月25日 土曜日 9時00分
生活タイムゾーン: 日本時間
```

LLM に UTC timestamp を渡さない。
LLM に offset 付き ISO timestamp を判断文脈の主要表現として渡さない。

LLM が扱うのは、日付、曜日、時刻帯、生活タイムゾーン名、相対時間の要約である。
コードが扱うのは、offset 付き timestamp、duration、deadline、cooldown、timeout である。

## duration と deadline

timeout、cooldown、経過時間、感情 decay は、timestamp の文字列表現ではなく duration と基準時刻から計算する。

少なくとも次を分ける。

| 種類 | 表現 |
|------|------|
| 起きた時刻 | offset 付きローカル timestamp |
| 未来の締切 | offset 付きローカル timestamp |
| 繰り返し予定 | local time と生活タイムゾーン |
| 経過時間 | `elapsed_seconds` |
| timeout | `timeout_ms` |

LLM には必要な要約だけを渡す。
たとえば `3時間前`、`今日の朝`、`明日の9時` のような表現を使う。

## 外部 timestamp

外部サービスや client から UTC または別 timezone の timestamp を受け取った場合、取り込み時に OtomeKairo のローカル時刻へ変換する。
元の timestamp が監査に必要な場合は source payload 側に残し、正本の判断時刻とは分ける。
