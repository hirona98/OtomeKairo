# 感情モデル

## 目的

OtomeKairo における感情は、会話の装飾でも人格設定の一部でもない。
感情は、経験に対する瞬間反応、現在の機嫌、対象への持続感情を分けて持ち、判断文脈を変える内部モデルとして扱う。

この文書は感情の正本設計を定める。
`memory/` 配下の文書は、感情についてはこの文書を参照する。

## 境界

感情モデルは、次と混同しない。

- 人格設定
  - 持続的な価値観、話し方、判断姿勢の基底
- 会話本文の表現補助
  - 顔タグや装飾表現などの出力演出
- `memory_units`
  - 継続理解や事実、関係、約束などの記憶本体

ここで重要なのは、「今どういう機嫌か」と「誰や何にどう感じているか」を同じ構造へ押し込まないことである。

## 正本構造

感情モデルの正本構造は、次の 3 つで構成する。

1. `episode_affect`
2. `mood_state`
3. `affect_state`

### `episode_affect`

`episode_affect` は、その判断サイクルで起きた瞬間反応のログである。

最小契約は次とする。

| 区分 | 項目 | 役割 |
|------|------|------|
| 識別 | `episode_affect_id` | 瞬間反応の識別子 |
| 所属 | `memory_set_id` | どの記憶集合に属するか |
| 根拠 | `episode_id` | どの `episode` に由来するか |
| 対象 | `target_scope_type / target_scope_key` | 何に対する反応か |
| 内容 | `affect_label` | 反応の短いラベル |
| 本文 | `summary_text` | 何に反応してどう揺れたかの短い本文 |
| 数値 | `vad` | `{v, a, d}` |
| 強度 | `intensity` | 反応の強さ |
| 根拠度 | `confidence` | 解釈の確からしさ |
| 観測時刻 | `observed_at` | その反応が起きた `episode` の時刻 |
| 保存時刻 | `created_at` | row を保存した時刻 |

制約は次とする。

- append-only で残す
- 1 `episode` あたり最大 4 件まで保存する
- 同じ `episode` の中では `target_scope_type / target_scope_key / affect_label` を 1 件に統合する
- `episode_affect_id` は一意にする
- `summary_text` は一意性や更新判定のキーに使わない

OtomeKairo は `1判断サイクル = 1episode` を基本単位にするため、瞬間反応も `event` ではなく `episode` に紐づける。

### `mood_state`

`mood_state` は、自分自身の現在の機嫌を表す単一状態である。

最小契約は次とする。

| 区分 | 項目 | 役割 |
|------|------|------|
| 識別 | `mood_state_id` | 現在機嫌の識別子 |
| 所属 | `memory_set_id` | どの記憶集合の機嫌か |
| 本文 | `summary_text` | 現在の空気感を表す補助本文 |
| 基調 | `baseline_vad` | ゆっくり変わる背景の機嫌 |
| 余韻 | `residual_vad` | 直近出来事の余韻 |
| 根拠度 | `confidence` | 現在の機嫌解釈の確からしさ |
| 観測時刻 | `observed_at` | 直近の数値更新に使った self 感情の時刻 |
| 作成時刻 | `created_at` | row の初回作成時刻 |
| 更新時刻 | `updated_at` | row の最終更新時刻 |

制約は次とする。

- `self` 専用で持つ
- `memory_set_id` ごとに常に 1 件だけを持つ
- `current_vad` は保存せず、読み出し時の派生値として計算する
- `summary_text` は一意性や数値更新のキーに使わない

`mood_state` は「いま少し機嫌がいい」「最近やや張りつめている」といった現在の個の空気感を表す。
複数の `affect_label` を並べて自己感情を増殖させない。

### `affect_state`

`affect_state` は、対象に向いた持続感情を表す状態である。

最小契約は次とする。

| 区分 | 項目 | 役割 |
|------|------|------|
| 識別 | `affect_state_id` | 持続感情の識別子 |
| 所属 | `memory_set_id` | どの記憶集合に属するか |
| 対象 | `target_scope_type / target_scope_key` | 何に向いた感情か |
| 内容 | `affect_label` | 持続感情の短いラベル |
| 本文 | `summary_text` | どういう傾向として育っているかの短い本文 |
| 数値 | `vad` | `{v, a, d}` |
| 強度 | `intensity` | 傾向の強さ |
| 根拠度 | `confidence` | その傾向の確からしさ |
| 観測時刻 | `observed_at` | 直近の支持 `episode` を観測した時刻 |
| 作成時刻 | `created_at` | row の初回作成時刻 |
| 更新時刻 | `updated_at` | row の最終更新時刻 |

制約は次とする。

- `relationship / topic / entity / world` など対象指向の感情だけを持つ
- `self` の現在機嫌は入れない
- 一意キーは `memory_set_id / target_scope_type / target_scope_key / affect_label` とする
- 同じ対象 `memory_set_id / target_scope_type / target_scope_key` では最大 2 件まで並存させる
- `summary_text` は一意性や更新判定のキーに使わない

`affect_state` は、「この相手には親しみが育っている」「この話題には警戒がある」といった持続傾向を表す。
単発反応をそのまま長期状態にしない。

## 旧設計からの置き換え

旧設計の `affect_state.surface / background` は採らない。
置き換えは次のとおりとする。

- 旧 `surface`
  - `episode_affect` と `mood_state.residual_vad`
- 旧 `background` のうち `self`
  - `mood_state.baseline_vad`
- 旧 `background` のうち対象指向感情
  - `affect_state`

この分離により、瞬間反応、機嫌、対象感情を別々に更新できる。

## 名称移行の注意

旧設計と新設計で、`affect_state` の意味は一致しない。

- 旧 `affect_state(target_scope_type=self, layer=surface)`
  - 新 `episode_affect(self)` と `mood_state.residual_vad`
- 旧 `affect_state(target_scope_type=self, layer=background)`
  - 新 `mood_state.baseline_vad`
- 旧 `affect_state(target_scope_type=relationship/topic/entity/world, layer=background)`
  - 新 `affect_state`

次の読み方は誤りとする。

- `self` の現在機嫌を `affect_state` と呼ぶこと
- `surface / background` を前提に新 `affect_state` を読むこと
- 旧 `affect_state` の語感のまま、新 `affect_state` に self 感情を入れること

## 数値モデル

OtomeKairo の感情数値は、最初から VAD を正にする。

- `vad`
  - `v` は快不快、`a` は覚醒、`d` は主導感を表す
  - 形式は `{v, a, d}`
  - 各軸の範囲は `-1.0 .. +1.0`
- `intensity`
  - 反応や傾向の強さ
  - 範囲は `0.0 .. 1.0`
- `confidence`
  - その感情解釈にどれだけ根拠があるか
  - 範囲は `0.0 .. 1.0`

ただし、VAD だけでは意味が足りない。
そのため、`episode_affect` と `affect_state` では `affect_label` と `summary_text` を必ず併用する。
`mood_state` では `baseline_vad` と `residual_vad` を保持し、現在の空気感は `summary_text` で補う。

## 更新責務

### `turn consolidation`

`turn consolidation` では、LLM が返す感情出力を `episode_affect` に限定する。

- LLM はそのターンの瞬間反応を構造化して返す
- コードは contract を検証して `episode_affect` を保存する
- コードは `target_scope_type=self` の `episode_affect.vad` から `mood_state` を deterministic に更新する
- `affect_state` の長期育成はここで確定しない

ここで重要なのは、LLM に長期感情状態の state 遷移を直接決めさせないことである。

### `mood_state` の更新原則

`mood_state` は次の原則で更新する。

- `baseline_vad` はゆっくり動かす
- `residual_vad` は即時に効かせる
- `residual_vad` は時間で減衰させる
- 強い単発反応だけで `baseline_vad` を急反転させない
- 現在の機嫌は常に単一状態として読む

これにより、「その場で少し揺れた」と「最近ずっと機嫌が悪い」を同じものとして扱わない。

deterministic 更新則は次に固定する。

前提関数:

- `clamp01(x)`
  - `0.0 .. 1.0` に丸める
- `clamp_vad(x)`
  - `v / a / d` の各軸を `-1.0 .. +1.0` に丸める
- `vad_lerp(cur, tgt, alpha)`
  - `clamp_vad(cur + alpha * (tgt - cur))`
- `vad_decay(vad, dt_seconds, half_life_seconds)`
  - `clamp_vad(vad * 0.5^(dt_seconds / half_life_seconds))`

固定パラメータ:

- `MOOD_BASELINE_HALFLIFE_SECONDS = 86400`
- `MOOD_RESIDUAL_HALFLIFE_SECONDS = 21600`
- `MOOD_RESIDUAL_ALPHA = 0.75`

ターン内集約:

1. `target_scope_type=self` かつ `target_scope_key=self` の `episode_affect` だけを集める
2. 各要素の重みを `weight_i = clamp01(intensity_i) * clamp01(confidence_i)` とする
3. `sum_weight = Σ weight_i` とする
4. `sum_weight == 0` なら、そのターンは `mood_state` の数値更新を行わない
5. `moment_vad = clamp_vad(Σ(weight_i * vad_i) / sum_weight)` とする
6. `moment_strength = clamp01(sum_weight)` とする
7. `moment_observed_at = max(observed_at_i)` とする

更新式:

1. 既存 `mood_state` がある場合は、`baseline_vad_prev / residual_vad_prev / previous_observed_at / created_at_prev` をその row から読む
2. 既存 `mood_state` が無い場合は、`baseline_vad_prev={0,0,0}`、`residual_vad_prev={0,0,0}`、`previous_observed_at=moment_observed_at`、`created_at_prev=write_time` を初期値とする
3. `dt_seconds = max(0, moment_observed_at - previous_observed_at)` とする
4. `alpha_base = clamp01((1 - 0.5^(dt_seconds / MOOD_BASELINE_HALFLIFE_SECONDS)) * moment_strength)` とする
5. `baseline_vad_new = vad_lerp(baseline_vad_prev, moment_vad, alpha_base)` とする
6. `residual_vad_decayed = vad_decay(residual_vad_prev, dt_seconds, MOOD_RESIDUAL_HALFLIFE_SECONDS)` とする
7. `residual_input = clamp_vad(moment_vad - baseline_vad_new)` とする
8. `residual_alpha = clamp01(MOOD_RESIDUAL_ALPHA * moment_strength)` とする
9. `residual_vad_new = vad_lerp(residual_vad_decayed, residual_input, residual_alpha)` とする
10. 保存値は `baseline_vad=baseline_vad_new`、`residual_vad=residual_vad_new`、`observed_at=moment_observed_at`、`created_at=created_at_prev`、`updated_at=write_time` とする

読み出し時の現在値:

- `elapsed_seconds = max(0, now - observed_at)` とする
- `current_vad = clamp_vad(baseline_vad + vad_decay(residual_vad, elapsed_seconds, MOOD_RESIDUAL_HALFLIFE_SECONDS))` とする

`current_vad` は inspection と判断入力で使う派生値であり、正本として保存しない。

### `reflective consolidation`

`reflective consolidation` では、複数ターンにまたがる反復から `affect_state` を育てる。

- 入力として最近の `episodes`、`episode_affect`、`memory_units`、`revisions` を見る
- 反復して現れる対象感情だけを `affect_state` に上げる
- 直近 1 件の強反応だけで長期感情を確定しない
- 長く支持が無い `affect_state` は弱めるか整理する

`affect_state` は「継続傾向」であり、「最新反応のキャッシュ」ではない。

## `AffectContext`

判断へ渡す内部感情文脈は `AffectContext` とする。
`AffectContext` は次を圧縮した内部コンテキストである。

- `mood_state`
  - 常に含める
- 関連 `affect_state`
  - 現在入力と `RecallHint` に関係する対象だけを含める
- 直近 `episode_affect`
  - 直近原因の追跡が必要な場合だけ少数含める

`AffectContext` は単独で判断を決める情報ではない。
`RecallPack`、現在入力、世界状態、能力状態を補う前景情報として扱う。

## inspection とデバッグ

inspection では、感情について少なくとも次を追えるようにする。

- 保存した `episode_affect` 件数
- `mood_state` の更新要約
- `affect_state` の更新要約
- 現在の `mood_state` の要約と、必要時だけ `baseline_vad / residual_vad / current_vad`
- 直近 `episode_affect` の要約

ここでも、生ログ全件や内部表全件を常に出す設計は採らない。
「なぜ今こういう機嫌か」「どの出来事が揺れの原因か」を追えることを優先する。

## やらないこと

次は採らない。

- `expression_addon` や顔タグを感情モデルの一部とみなすこと
- `self` の機嫌を複数の `affect_state` 行へ分散させること
- 単発反応をそのまま長期感情へ昇格させること
- 感情だけで `memory_units.confidence` を上げること
- 感情文脈だけで判断を決めること

## 実装移行の基準

感情実装は、この文書に合わせて次の形へ寄せる。

1. 同期更新の出力契約を `episode_affect` 中心へ切り替える
2. `mood_state` を deterministic 更新へ分離する
3. `affect_state` の育成を `reflective consolidation` 中心へ寄せる

この文書は旧 `affect_updates -> affect_state` 設計を置き換える正本とする。
