# 設定と状態

<!-- Block: Role -->
## この文書の役割

この文書は、OtomeKairo が何を設定として保持し、何を状態として扱うかを定める正本である。

ここで固定したいのは次の内容である。

- OtomeKairo 側が正本として保持する設定項目
- 永続設定と `runtime_state` の境界
- `CocoroConsole` が編集できる対象とできない対象
- MVP で設定に含めるものと含めないもの

API パスや保存形式はここでは定めない。
ただし、設定責務と状態境界はここで固定する。

<!-- Block: Premise -->
## 基本前提

OtomeKairo では、設定の正本は常にサーバ側に置く。
`CocoroConsole` はその設定を編集する UI であり、設定値の正本にはならない。

また、状態はすべて同じ層で扱わず、次のように分ける。

- 永続設定
  - 明示操作で変える正本
- `runtime_state`
  - 現在のプロセスがどう動いているかを表す稼働状態
- `world_state`
  - 次の判断にも持ち越したい外界状態
- `current_observation`
  - その場限りの生観測
- `capability_state`
  - 能力ごとの実行可否や制約

ここで重要なのは、「長く残るもの」だけで設定に入れるのではなく、「明示操作で変更される OtomeKairo 自身の条件」だけを設定に入れることである。

<!-- Block: Settings -->
## MVP で保持する設定

MVP では、次を OtomeKairo の永続設定として保持する。

- `selected_persona_id`
- `selected_memory_set_id`
- `wake_policy`

この 3 つを最小セットにする理由は次のとおりである。

- 人格と記憶の独立切り替えが MVP の中心機能である
- 自発会話の頻度は OtomeKairo 側の設定として保持する必要がある
- それ以外の項目まで広げると、API と UI の責務が先に膨らむ

<!-- Block: PersonaMemorySelection -->
## `selected_persona_id` と `selected_memory_set_id`

`selected_persona_id` と `selected_memory_set_id` は、永続設定の正本として保持する。

これは「現在の選択内容」を表す値ではあるが、意味としては OtomeKairo の設定である。
理由は次のとおりである。

- 明示的な切り替え操作でのみ変わる
- 再起動後も同じ選択を引き継ぐべきである
- `CocoroConsole` はその選択を編集するだけで、選択の正本にはならない

一方で、判断時に参照される `runtime_state` には、現在読み込み済みの人格設定参照と記憶セット参照の反映状況を含めてよい。
ただし、そこで見える値は「設定がいま指していて、実際に反映されている対象」であり、正本そのものではない。

<!-- Block: WakePolicy -->
## `wake_policy` の扱い

`wake_policy` も永続設定として保持する。

MVP では、自発性はまず会話主体に限定するため、`wake_policy` も会話起点の判断機会だけを制御する設定として扱う。

最小契約は次でよい。

| 項目 | 意味 |
|------|------|
| `mode` | `disabled` または `interval` |
| `interval_minutes` | `mode=interval` のときの起床間隔 |

この段階では、複雑な時間帯制御や多段ポリシーは入れない。
まずは「自発判断を止めるか」「一定間隔で判断機会を与えるか」だけを固定する。

<!-- Block: NotSettings -->
## 設定に含めないもの

MVP では、次は永続設定に含めない。

- `connection_state`
- `ongoing_action_exists`
- 現在地や前景アプリのような外界条件
- 能力の一時的な可用性
- 想起や更新の内部処理で使う一時値

これらは、設定ではなく現在状態または内部処理状態である。

また、モデル役割の分け方や実際のモデル指定も、MVP では `CocoroConsole` から編集する設定に含めない。
これは会話品質やコスト設計に強く結びつくため、当面は設計またはデプロイ設定として扱う。

<!-- Block: RuntimeState -->
## `runtime_state` に入れるもの

`runtime_state` は、OtomeKairo 自身の現在の稼働条件だけを持つ。

MVP では、少なくとも次を入れてよい。

- 現在読み込み済みの人格設定参照
- 現在読み込み済みの記憶セット参照
- 起床スケジューラが稼働中かどうか
- `connection_state`
- `ongoing_action_exists`

ここでの「現在読み込み済み参照」は、永続設定が指している対象として現在反映中の人格設定や記憶セットを指す。
この段階では、その表現を ID に固定しない。
内部参照、読み込み済み実体の要約、またはそれらに準ずる軽量表現で持ってよい。
つまり、設定の正本は永続設定側にあり、`runtime_state` はその現在反映中の参照または実体要約を表す。

<!-- Block: EditableBoundary -->
## `CocoroConsole` が編集できる対象

MVP で `CocoroConsole` が編集できる対象は次に限定する。

- `selected_persona_id`
- `selected_memory_set_id`
- `wake_policy`

逆に、次は `CocoroConsole` の編集対象にしない。

- `runtime_state`
- `world_state`
- `capability_state`
- モデル選定や内部ログ詳細
- 記憶の内部テーブルそのもの

この境界により、`CocoroConsole` は編集 UI でありつつも、OtomeKairo 内部の判断責務や実行責務を侵食しないで済む。

<!-- Block: DesignDecision -->
## ここで固定したい設計判断

この文書で固定したいのは次である。

- `selected_persona_id` と `selected_memory_set_id` は永続設定の正本として持つ
- `wake_policy` は自発会話の判断機会を制御する永続設定として持つ
- `runtime_state` は設定の正本ではなく、現在の反映状態と稼働状態を表す
- `CocoroConsole` は設定を編集できるが、状態の正本や判断主体にはならない
- モデル関連設定は MVP では公開設定に含めない
