# 目的とMVP

## 目的

OtomeKairo の目的は、自分の意志で行動し、人間とともに生活する、個として独立した存在を作ることである。

## 目指す存在像

この存在像には次の性質を含む。

- 個々の人物や出来事を認識し、世界を自分なりに理解する
- 継続する人格と記憶を持つ
- 記憶によって考え方や反応が変わる（記憶が人格を構成する）
- 移動可能で五感を持つ
- インターネットアクセスが可能

## MVP の範囲

最初の MVP は、会話を中心とした記憶と人格の核を成立させることに集中する。

MVP で含めるもの:

- `CocoroConsole` を唯一のクライアントとして接続する
- OtomeKairo を常駐サーバとして稼働させる
- ユーザー文字入力と軽量なクライアント文脈を観測として受け取る
- 会話を継続し、日をまたいだ記憶を保持する
- 自発的に会話提案できる
- ユーザーの嗜好を学習できる
- 同一人格のまま複数の記憶を切り替えて検証できるように、人格と記憶を独立に切り替えられる

MVP でまだ主対象にしないもの:

- カメラ、移動手段、Web アクセスなどの外界行動そのもの
- 人間が使うあらゆる接点への展開
- 細かな安全制御や権限制御

ただし、将来の拡張を見据えて、会話以外の行動も収容できる設計方針はこの段階で定める。

## 成功条件

MVP の成功条件は次の 3 点とする。

- 数日またいでも、別セッションで過去を踏まえた会話ができる
- 自分から次の行動や会話を提案できる
- ユーザーの嗜好を学習し、それが会話に反映される

## 非目標

現段階では、次を目標にしない。

- すべての行動能力を一度に実装すること
- 人間と同等の身体性を先に作ること
- 実装都合で仕様を先に決めること
