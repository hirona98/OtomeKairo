# 現行計画

## この文書の役割

この文書は、OtomeKairo の現在地、完了済み範囲、残っている拡張テーマだけを簡潔に管理する。

- 設計判断は `docs/design/` を正とする
- 実装状態はコードを正とする
- 実装や仕様を変えたら、同じ変更内で docs も更新する
- 旧仕様互換だけの移行レイヤーや検索フォールバックは扱わない

## 現在地

現在の実装は、主要な会話基盤と観測基盤までは通っている。

- 会話基盤:
  通常会話 1 サイクル、bootstrap、設定取得、設定変更、catalog、inspection API
- 記憶基盤:
  `SQLite + sqlite-vec`、`turn consolidation`、`RecallHint`、`memory_interpretation`、`RecallPack`、`reflective consolidation`
- 自発判断基盤:
  runtime-only の `pending_intent` キュー、wake observation API、background 起床スケジューラ、`reply / noop / pending_intent`
- 観測基盤:
  `desktop_watch` 第一段、`client_context` の正規化、`vision.capture` capability、`vision.capture` 対応 client 自動選択、`logs/stream`
- 後段処理:
  durable な memory postprocess worker、`status.runtime_summary`、inspection の `memory_trace`
- 運用確認:
  `scripts/run_long_smoke.py` による `smoke / soak` 実行、isolated data dir、主要な capture failure case の確認
- 意味判断での LLM 採用:
  `reflective consolidation` の `summary_text`、`event_evidence`、`RecallPack` の意味的選別、pending-intent 候補選択

この前提では、会話、記憶、自発判断、`desktop_watch` 方針と `vision.capture` capability の第一段、長時間 smoke までは完了している。
以降は、完成形との差分を埋める拡張段階として扱う。

## 既知の制約

次は現時点の設計上の制約であり、未整理のバグ一覧ではない。

- `event_evidence` は短い圧縮根拠であり、逐語引用や正確引用は保証しない
- `RecallHint` は長期記憶を読まず、現在入力と直近文脈だけから作る
- `RecallHint` は `interaction_mode` と `primary_recall_focus` を主軸にし、完全な同時最適化は行わない
- 通常会話 API は `text` と `client_context` だけを受け、画像入力をまだ持たない
- `desktop_watch` は `vision.capture` の image payload を取得できるが、まだ画像内容自体は判断に入れず、`client_context` 主体で扱う
- background wake は複雑な時間帯制御や外界行動実行をまだ持たない
- 回帰テスト整備は現行計画の完了条件に含めない

## 次の拡張

現時点で保留している主要な拡張は次である。

- 通常会話で画像入力を受け、共有判断パイプラインへ入れられるようにする
- `vision.capture` の image payload を判断材料へ入れる
- 時刻帯や前景状況を使う判断補助を絞って足す
- `--profile soak` を土台に、より長い実時間運用と追加 failure case を確認する

上の拡張は後続の拡張テーマとして扱い、現行基盤の完了扱いとは分ける。
