# AGENTS.md

## このリポジトリの目的
- LLMを使用して人間を模した自立可能なAIを完成させる

## 方針
- ユーザーとのコミュニケーションには日本語を使う
- シンプルさを優先すること
- 要件や設計の判断は docs ディレクトリを正として扱うこと
- 同じ仕様を複数の docs に正本として書かないこと。意味境界、API wire、実装計画の正本を分け、他の docs ではリンク参照に留めること
- docs とコードは常に同期させること
- 修正時は、本来あるべき姿への修正を行うこと（最小限の修正にしない）
- 旧仕様との互換レイヤーや移行用コードを入れないこと
- フォールバック処理を入れないこと
- 仕様が固まるまでは、新規テストは原則として実装しない
- コメントを書くこと
- ログ、サンプル、テストデータに秘密情報を含めない

## ツール運用
- Gitのユーザ名とメールアドレスを変更しない
- ユーザーが明示的に指示するまでコミットしない
